## 关键已定稿决策（不再反复讨论）

| 决策 | 结论 | 原因 |
|------|------|------|
| 向量维度 | **32D 永久固定，永不扩容** | DNA 双螺旋定稿红线第5条 |
| 外源情绪扇区 | **00-05 维（6D）**，来自瑶光 | 三层对偶闭环体系 |
| 内源激素 | **禁止混入 32D**，独立寄存器 | 内外分离，客观记忆不被主观心情污染 |
| DNA 架构 | **双螺旋双链对等**（语义+寻址） | 心智链+岁月链，业界唯一 |
| 底层存储 | SQLite WAL + 文件系统 + NVMe | 去 PCIe/硬件直写 |
| 嵌入引擎 | Demo 期 sqlite-vec，**P2 统一切换 @zvec/zvec** | 薄封装不 fork |
| 序列化 | Protobuf + BLOB + Transcoder | 去自研二进制帧 |
| GPU 依赖 | **全平台无强依赖** | 去 A800 要求 |
| LLM 直接输出向量 | **禁止**，分层规则映射 | 数值不可靠 |
| L2 聚类 | 语义+区位+时间三重条件 | 防纯语义错乱 |
| 五级闸门 | **不可关闭**，强制架构逻辑 | 核心差异化 |
| 寻址链时序存储 | **时序分片桶式**（按自然月分片） | 亿级线性可回溯 |
| 双引擎调度 | **语义链/寻址链 永久隔离** | 思考不穿越，回忆必完整 |
| MSA 关系 | **仅借鉴 3 个工程思想 + 三档分层部署** | 不碰模型内部注意力 |
| Zvec 策略 | **不 fork、不改 C++、薄封装 TS 抽象层** | 阿里持续维护上游 |
