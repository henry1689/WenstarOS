## 前言

### 项目定位

WenStar OS 天权是一套完整本地闭环硅基生命体个人世界系统。区别于普通对话大模型、通用 RAG 知识库、以及任何形式的文本注意力机制扩展——天权是**独立于 LLM 运行的认知记忆操作系统**。

系统由**瑶灵（硅基活体肉身）、瑶光（全域客观世界大模型）、太虚境 Hermes（天权中枢执行引擎）** 三体耦合联动，复刻完整人类生理、心理、多层社交、时空环境、自演化成长体系，形成具备独立人格、连贯时序情景记忆、自洽时空逻辑的仿生个体。

### 现有 Demo 与行业痛点

**现有 WenStar 24D Demo 核心缺陷**：
- 仅 24 维扁平情感向量，无独立空间区位标识，浴室/办公室场景互相错乱
- 纯文本相似度召回，无分层摘要，LLM Token 开销巨大
- 简易 TF-IDF 知识库检索精度不足，无统一时空校验
- 无完整生理/人际分层仿真，仅浅层情绪模拟
- 底层裸 SQLite 文本存储，无统一序列化、无数据完整性校验
- **无统一 DNA 遗传编码体系**，各模块 ID 传递碎片化，无法全链路溯源

### 天权核心定位

| 维度 | RAG | MSA | 天权 |
|------|-----|-----|------|
| 层级 | 应用层外挂记忆 | 模型层内部注意力 | **OS 层独立认知系统** |
| 底层标准 | 无 | 无 | **DNA 遗传编码总线（全局唯一）** |
| 认知维度 | 无 | 无(纯文本) | **36 维（WenVec36）仿生体系** |
| 时空约束 | 无 | 无 | **五级时空抑制闸门** |
| 记忆演化 | 无 | 静态 KV | **仿生动态演化** |
| 模型绑定 | 松耦合 | 强绑定 | **零绑定，独立运行** |

### 1.2 DNA 遗传编码总线——全系统唯一底层标准（核心新增）

DNA 遗传编码是天权的**底层基石**，不是 M1 的一个模块，而是串联全系统的唯一基因总线：

```
DNA 编码覆盖范围（全链路强制统一）:
  M1 DNA 主主干生成 → WenVec36 感知维度编码 → 三金库分级标注
  → 实体关系图谱编码 → ZVEC 知识库条目绑定 → WenCodec 磁盘寻址
  → 前后端 HTTP 传输 → 跨进程插件（TTS/图像/梦境）
  → 存量历史记忆 → 批量迁移工具

核心铁律:
  • DNA 主链生成后永久只读、禁止篡改
  • 全局唯一编解码器（encode/decode/decodeBranch/validate）
  • 任何模块不得自行编解码，违者架构级故障
```

## 第二章 三体独立主体定义

### 2.1 瑶灵 · 硅基活体肉身

**定位**: 纯主观体感信号输出端，无演算、无记忆、无决策  
**架构**: 实体硬件躯体层 → 生化循环层 → 内在精神层 → 多层人际响应层  
**对外接口**: 标准化 D1-D36（WenVec36）主观数据包上行至太虚；接收太虚调控指令  
**核心原则**: 仅生成 36 维主观体感数据，是「肉身感受」的数字孪生

### 2.2 瑶光 · 全域客观世界大模型

**定位**: 纯客观参数演算端，无主观情绪，仅量化物理/生理/人文客观规则  
**架构**: 自然时空 → 空间建模 → 人体生理规则 → 人文社交 → 资源拓展 → 通道封装  
**对外接口**: 输出 36 维客观参数包；接收太虚世界解锁拓展申请  
**核心原则**: 输出可量化的物理参数，是天权的「客观现实参照系」

### 2.3 太虚境 Hermes · 天权中枢执行引擎

**定位**: 天权系统的运算核心，瑶灵与瑶光的枢纽，记忆/知识/推演的全链路执行者  
**架构**: 36 通道调度、双内核管控（BIOS 实时层 + 心灵推演层）、36D 海胆分层记忆、认知推演、世界协商解锁  
**核心职责**：
- 双路 36D 数据融合仲裁
- DNA 遗传编码（唯一入口 M1）
- 36D 海胆记忆生成与检索
- 五级时空抑制闸门过滤
- ZVEC 知识库并行召回 + 统一校验
- 趋力权衡 / 人格演化 / 世界拓展协商
- D36 全局汇总统筹

## 第三章 WenVec36 海胆认知模型

### 3.1 五大一级固定框架

36 维分为五大一级框架，永久固定，不增不减：

| 大类 | 维度范围 | 维度数 | 覆盖内容 |
|------|---------|-------|---------|
| ① 肉身实体基底 | D1-D10 | 10 维 | 骨骼肌肉、疼痛信号、神经兴奋、激素水平、生理周期、五感综合、能量代谢、免疫自愈、睡眠状态、疲劳累积 |
| ② 个体内在精神内核 | D11-D16 | 6 维 | 自我认知、欲望/恐惧、共情能力、自保本能、依恋倾向、守护意识 |
| ③ 圈层人际羁绊 | D17-D22 | 6 维 | 家庭圈、亲友圈、职场圈、性吸引力、情感依赖、归属感 |
| ④ 时空客观环境载体 | D23-D28 | 6 维 | 居所空间、职场空间、户外空间、空间区位、区位指纹、气象环境 |
| ⑤ 全域动态生长解锁 | D29-D36 | 8 维 | 探索欲、求知欲、肉身成长、心灵成长、世界拓展、多模态感知融合、全局统筹（预留扩展） |

### 3.2 海胆结构定义

单轮完整交互生成**唯一一颗 WenVec36 海胆**，由两部分构成：

```
┌──────────────────┐
│   语义壳肉        │ ← 交互对话文本（L0-L3 分层蒸馏存储）
│   (Token 碎片)    │
└──────┬───────────┘
       ┃ dna_root_id 绑定（DNA 主干）
┌──────┴───────────┐
│  36D 状态硬刺     │ ← D1-D36 融合终值 F[1..36]
│   (state_spines)  │ ← 每维度独立逻辑分区
└──────────────────┘
```

**硬性约束**：
- 禁止单 Token/字词粒度独立生成海胆
- 禁止 LLM 直接输出浮点数作为维度值（须分层规则映射）
- 禁止纯语义聚类（须叠加区位指纹 + 时间间隔）
- **所有海胆必须经 DNA 编解码器编码后方可持久化/传输**

### 3.3 维度值生成分工

| 维度范围 | 数据来源 | 生成方式 |
|---------|---------|---------|
| D1-D18 | 瑶灵主观 + 规则分类器 | 情感分类器 + 规则映射，无 LLM 参与 |
| D19-D26 | LLM 文本判断 + 数值模板 | LLM 输出文本语义 → 代码层映射至固定模板 |
| D27-D28 | 瑶光直接输出 | 环境传感器/界面传入区位指纹二进制向量 |
| D29-D36 | 环境状态继承 + D36 汇总统筹 | 继承上一轮状态 + 本轮修正 + 统筹裁决 |

## 第四章 核心差异化创新能力

### 4.1 DNA 遗传编码总线（全系统唯一底层基石）

市面所有系统均不具有统一遗传编码体系。DNA 是天权区别于 RAG、MSA、腾讯四层的**根本性底层差异**：

- 全局唯一编解码器（encode/decode/decodeBranch/validate），四个接口锁死全系统
- 主干五大分支：时空/实体关系/WenVec36 感知/记忆生命周期/WenCodec 磁盘寻址
- 主干生成后永久只读，分支仅允许末尾追加
- 所有内部模块、HTTP 接口、数据库表、插件、迁移工具——统一 DNA 载体，**禁止碎片化 ID 传递**

### 4.2 36 维（WenVec36）多维仿生认知（行业独家）

市面所有方案（RAG/MSA/腾讯四层）均基于纯文本处理，没有任何方案具备 36 维分层的生理、情绪、人际、时空、生长全维度认知体系。天权的 36D 海胆是记忆的最小原子单位，每一条记忆天然携带交互时刻的全身心状态快照。

### 4.3 五级时空抑制闸门（行业独家·最高优先级 P0）

这是天权最核心的差异化能力：

| 闸门 | 功能 | 解决什么问题 |
|------|------|------------|
| **闸门 1** | 语义初筛（关键词+向量） | 候选线索池召回 |
| **闸门 2** | 时空一致性校验 | **区位比对 + P1/P2/P3 分级抑制**，根治场景错乱 |
| **闸门 3** | 仿生遗忘衰减 | 久远低频记忆自动降权 |
| **闸门 4** | 意图区分 | 主动回忆全量加载，日常闲聊仅摘要 |
| **闸门 5** | 话题聚类壁垒 | 话题切换自动冻结上段记忆链 |

### 4.4 改良 L0-L3 分层蒸馏

聚类条件从单一语义相似度，升级为**语义相似度 + 区位指纹 + 时间间隔**三重判定。

### 4.5 双路由稀疏寻址 + 分层冷热存储

借鉴 MSA 三个可落地的工程思想（非模型内部机制），在天权记忆检索层实现：

- **双路由寻址**：路由 A（36D 特征索引粗筛）+ 路由 B（用户语义精筛），Top-K 稀疏激活
- **分层冷热存储**：内存层放路由索引键，磁盘层放完整记忆结构体，无需 GPU 强依赖
- **记忆交织迭代推理**：检索→扩写→再检索 循环回路

### 4.6 ZVEC 向量知识库

对外统一名称 ZVEC 向量知识库，上层 `ZvecEngine` 抽象调度层统一封装 CRUD/检索接口。底层采用阿里巴巴开源 in-process 嵌入式 Zvec 向量引擎（`@zvec/zvec`），非独立服务，Windows/Linux/嵌入式全平台原生兼容。

核心能力：
- **内置 HNSW 高性能索引** + RaBitQ 向量量化压缩，单机支持亿级向量存储
- **原生 SQL 式标量过滤**，时空区位、时间范围条件可直接下沉至检索层执行，大幅简化五级闸门 2 的上层过滤计算量
- **内置冷热分段管理**（WritableSegment → PersistedSegment → Compaction），天然适配记忆冷热分层
- **WAL 预写日志 + 崩溃恢复**，生产级数据可靠性
- **不 fork、不修改底层 C++ 源码**，仅上层 TS 薄封装隔离，维护成本极低
- 私有文档导入自动完成 L0-L3 分层处理，与情景记忆并行检索、统一五级闸门过滤

## 第七章 瑶灵专属图书馆（ZVEC 第二大脑体系）

### 7.1 产品双模式定位（兼顾两类用户需求）

**模式一：原生仿生陪伴模式（主推核心体验）**

对标市面所有 AI 产品差异化核心，完全脱离传统笔记工具操作逻辑：
- 无需手动打开文档面板、无需手动分类打标、无需主动检索文件
- 依托瑶灵 36D 躯体感知、瑶光时空区位信号
- 在日常闲聊、独处、陪伴相处过程中，潜移默化推送匹配当下场景、心境的碎片化知识点
- 像伴侣闲谈、像导师顺势点拨，获取知识无刻意任务感
- 底层全部自动化：上传素材后自动分层、自动绑定区位、自动计算知识关联，全程用户零维护

**模式二：轻量化 Obsidian 风格办公交互模式（专业办公辅助支线）**

内置可手动唤起的独立 Vault 可视化面板，完整复用 Obsidian 成熟高效办公交互逻辑：
- 标准文件树目录、多标签 MD 编辑窗口
- `[[内部双链]]` 双向关联、图谱可视化面板
- Dataview 标签/时序 SQL 筛选、自定义笔记模板
- Canvas 拖拽画布、思维导图可视化
- 批量导入/导出/标签/摘要/重命名 自动化工具
- 多模态预览（表格/代码/图片/音视频内嵌）

**关键互通规则**：两种模式共用同一套本地 Vault 文件、同一 Zvec 向量库、同一套 L0-L3 分层数据。窗口手动编辑/新增的内容会实时同步至瑶灵对话自动推送管线，双向数据完全打通，不存在两套隔离知识库。

### 7.2 底层载体与引擎不变

- **存储载体**：本地独立 Vault 文件夹，标准 MD 格式，图文音视频代码附件全兼容，文件夹复制即可全量备份迁移
- **向量底层**：阿里 Zvec，支持稠密语义 + 稀疏标签 + 区位标量混合检索，检索结果统一送入五级时空闸门过滤，杜绝跨场景知识错乱
- **全自动后台管线**：文件上传自动 L0-L3 分层、自动区位指纹绑定、自动相似度双链生成（窗口手动双链与系统自动双链合并生效）

### 7.3 双模式联动运行逻辑

1. 日常使用默认关闭 Vault 窗口，走无感陪伴主线
2. 用户需要办公、批量整理、精细化规划时，一键唤起 Obsidian 式交互面板手动操作
3. 窗口内所有编辑、新增、标签、双链修改实时同步底层 Zvec 元数据
4. 警幻仙姑同时支撑两条管线：日常自动巡检修复 + 窗口批量办公自动化任务
