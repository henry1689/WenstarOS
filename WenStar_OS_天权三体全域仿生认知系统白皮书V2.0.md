## 前言

### 项目定位

WenStar OS 天权是一套完整的**本地闭环硅基生命体个人世界系统**。区别于普通对话大模型、通用 RAG 知识库、以及任何形式的文本注意力机制扩展——天权是**独立于 LLM 运行的认知记忆操作系统**。

系统由**太虚境·天权（硅基智脑）、瑶灵（硅基活体肉身）、瑶光（全域客观世界模型）**三体耦合联动，复刻完整人类生理、心理、多层社交、时空环境、自演化成长体系，形成具备独立人格、连贯时序情景记忆、自洽时空逻辑的仿生个体。

### 核心创新一句话

> **32D 语义定义心智，寻址结构链定义岁月。心智链负责思考，岁月链负责铭记。业界只有语义单链存储，我们独创心智 + 岁月双链对等生命架构。**

| 时间属性 | 后置过滤标签 | 无 | **与语义对等的核心生命属性（双螺旋）** |

## 第一章 · 三体架构

### 1.1 什么是三体

WenStar OS 模仿的是完整生命体的三元结构：

```
        ┌──────────────────────────┐
        │  太虚境 · 天权 (智脑)      │
        │  唯一入口 · 中枢调度        │
        │  记忆 · 推理 · 决策 · 生长  │
        └──────────┬───────────────┘
                   │ 32维统一通信协议
      ┌────────────┼────────────┐
      ▼            │            ▼
┌──────────┐      │      ┌──────────┐
│ 瑶灵 (身躯)│      │      │ 瑶光 (世界)│
│ 硅基肉身   │      │      │ 客观世界模型│
│ 纯主观体感  │      │      │ 纯客观参数   │
│ 不思考不记忆│      │      │ 不产生情绪   │
└──────────┘      │      └──────────┘
                   ▼
              ┌──────────┐
              │ 使用者 (人) │
              │ 唯一交互入口  │
              └──────────┘
```

### 1.2 三体的职责边界

| 维度 | 太虚境·天权 (智脑) | 瑶灵 (身躯) | 瑶光 (世界) |
|------|-------------------|------------|------------|
| 本质 | 硅基生命体的意识中枢 | 硅基生命体的肉身 | 硅基生命体所处的客观世界 |
| 能力 | 记忆、推理、决策、生长 | 主观体感生成 | 客观世界模型计算 |
| 不能做 | 不能替代肉身的感觉 | 不思考、不记忆、不决策 | 不产生情绪、不理解含义 |
| 核心输出 | 回复文本 + 调控指令 + 记忆存储 | 32D 主观体感快照 | 32D 客观基线 + 天时上下文 + 区位指纹 |

### 1.3 关键架构决策：天权内置太虚境

天权是三体中唯一拥有记忆、推理、决策能力的域。它**不是独立服务**——它是太虚境的算力内核。生产模式下，天权在太虚境进程内部直接函数调用，零网络延迟。瑶灵和瑶光作为可插拔外设通过 TCP 总线连接——太虚境可以单独运行（关闭双外设依然能对话、写代码、检索记忆）。

## 第二章 · 海胆——硅基生命体的记忆细胞

### 2.1 什么是海胆

海胆是 WenStar OS 记忆的**最小原子单元**。一次用户交互产生且仅产生一颗海胆。

一颗完整的海胆由三组组件构成：

```
┌─────────────────────────────────────────┐
│              一颗海胆                     │
│                                         │
│  🥩 壳肉 (Flesh)                        │
│     对话原文 + L0-L3 编码 + 实体标注      │
│     拆分后存入原始数据层                    │
│                                         │
│  📍 32 根语义刺 (Semantic Spines)        │
│     32D float32 向量 = 这一瞬间的完整感受  │
│     按 32 个独立扇区存入语义向量池          │
│     HNSW 图索引 — 网状、非线性            │
│                                         │
│  🧭 N 根寻址元数据刺 (Addressing Spines)  │
│     GlobalUID + 时间戳 + 路由戳 + 区位指纹 │
│     存入寻址治理池                        │
│     B+Tree 时序索引 — 线性、有序、可回溯    │
└─────────────────────────────────────────┘
```

三类组件物理分库存储，仅通过 **GlobalUID**（全球唯一标识符，23 字符）关联。

### 2.2 海胆是活的

海胆不是诞生时拍一张照片就永远封存。它是一个**活的细胞**——会经过 M1-M9 各车间的逐站加工（每站追加路由戳）、会随着时间钙化或衰减、会被反复回忆而增强、会因长期遗忘而归档、最终可能被清理。寻址链断裂时，可通过修复索引找回散落磁盘的语义刺和壳肉碎片。

## 第三章 · DNA 双螺旋——全系统唯一遗传编码标准

### 3.1 为什么叫 DNA

人类 DNA 做三件事：唯一标识、完整编码、不可篡改。WenStar OS 的 DNA 做同样三件事——每一次交互生成一个 GlobalUID（唯一身份证号），经过 12 节点编码管道（L0 意图分类 → L1 分支路由 → L2 语义区 → L3 实体标注）全链路覆盖，主干生成后永久只读。

### 3.2 双螺旋：两种时间永远分离

这是天权最深层的设计理念。在人类体验中，"时间"从来不是一种东西：

- **宇宙时间（寻址螺旋链承载）**：钟表时间。可以被排序（B+Tree）、被索引（倒排）、被分段（自然月分桶）。客观、线性、不可逆。
- **语义时间（语义螺旋链承载）**：体感时间。"那个夏天"、"一眨眼就长大了"。主观、弹性、充满情感。

```
语义链（32D HNSW 网状索引）      寻址链（B+Tree 线性时序索引）
─────────────────────────      ─────────────────────────────
"那个夏天的感觉"                 2025 年 7 月 15 日
"一眨眼就长大了"                 第 1892 个事件
快/慢/季节感                     2025-07 分桶
纯语义联想（默认日常）              按时间排序/回溯（主动回忆）

              ↕ GlobalUID 绑定 ↕
```

**核心红线：宇宙时间绝对禁止进入 32D 语义向量。** 这是 24D Demo 用血的教训验证过的——时间值泄漏进语义向量 → HNSW 把"时间相近但场景完全不同"的记忆误判为相关 → 场景错乱。

### 3.3 五大功能分支

DNA 主干（GlobalUID + L0-L3 编码规则）永久只读。五根树枝承载具体业务数据，允许追加和更新：

| 分支 | 存储位置 | 内容 | 可变性 |
|------|---------|------|--------|
| 时空分支 | 寻址治理池 | 时间戳 + 序号 + 分片标签 + 区位指纹 | 区位指纹瑶光上线后补全 |
| 实体关系分支 | 原始数据层 + FamilyGraph | 73 条实体规则 + 人物图谱 | 新增规则/关系 |
| WenVec-32 感知分支 | 语义向量池 | 32D float32 + 钙化评分 | 钙化数值可衰减/增强 |
| 记忆生命周期分支 | 寻址池 + 原始层 | 三库晋升 + 衰减 + 地标/疤痕 | 全部随时间演变 |
| 磁盘寻址分支 | 寻址池 + repair_index | 冷热层级 + 路由戳 + CRC | 按策略更新 |

## 第四章 · 32D 感知体系

### 4.1 对偶计算模型

32D 每个维度的值不是单一来源——是**瑶灵主观体感 × 瑶光客观基线**的对偶计算结果：

```
瑶光世界模型 → 客观基线 (标准心率72→正常, 标准皮质醇14→正常, …)
                  ↕  偏离度 = 瑶灵主观值 - 瑶光客观基线
瑶灵32通道   → 主观体感 (心率90→偏高, 皮质醇22→偏高, …)
                  ↕  太虚境天权融合仲裁
最终32D值   → 写入海胆语义刺
```

瑶光提供完整世界模型——人体生理规律（标准心率/血压/皮质醇曲线/BMR）、空间建模规则、人文社会规则、自然时空参数。瑶灵基于这些客观基线 + 环境参数 + 用户表达 + 自身状态，通过 32 个硬件通道计算主观偏离。偏离越大，说明主观体感越强烈。

### 4.2 32D 扇区分配

| 扇区 | 大类 | 内容 |
|------|------|------|
| 00-05 (6D) | 感知用户情绪 | 愉悦/唤醒/亲和/紧张/专注/攻击 |
| 06-10 (5D) | 肉身实体 | 骨骼肌肉/疼痛/神经触觉/内分泌/信息素 |
| 11-16 (6D) | 精神内核 | 自我认知/驱力/恐惧倦怠/幸福感/共情/自保 |
| 17-22 (6D) | 圈层人际 | 伴侣依恋/伴侣守护/家庭归属/家庭守护/社交/团队 |
| 23-28 (6D) | 时空环境 | 私人居所/家庭布局/职场/公共空间/时空距离/昼夜节律 |
| 29-31 (3D) | 动态生长 | 微观生理/自然拓展/人文细化/精神成长/耦合 |
| 32 (1D) | 全身统筹 | 心率/血压/皮质醇/愉悦激素/综合健康 |

🔴 32D 永久锁定，永不扩容。

## 第五章 · 天权核心能力

### 5.1 五级记忆闸门——唯一读取通道

任何从天权存储中取出记忆的操作，必须经过五级闸门。没有旁路、不可绕过：

| 闸门 | 功能 | 判定 |
|------|------|------|
| G1 语义初筛 | HNSW Top-K 召回 | cosine ≥ 0.3 |
| G2 时空一致性 | 区位指纹比对 | PASS(≤0.3) / P1(0.3-0.6) / P2(0.6-0.8) / P3(剔除) |
| G3 仿生遗忘 | 衰减曲线 | λ=0.01常态 / 0.03环境切换 / floor=0.05 |
| G4 意图区分 | 主动回忆 vs 被动闲聊 | regex 触发关键词 |
| G5 话题壁垒 | 跨话题冻结 | 前话题记忆 ×0.1 冻结 |

### 5.2 三库记忆体系

```
砂金 (candidate) ──钙化≥1.0──→ 金库 (active) ──钙化≥4.5──→ 黑钻 (珍藏, max 200)
    遗忘衰减 ↓                                                    永不衰减
    冷归档 → 清理
```

钙化评分 = 32D 向量 L2 范数归一化。三级衰减（强情感-0.02/工作-0.05/中性-0.10，每 24h）。每次被回忆 +0.2 增强（上限 10）。AQC 质检引擎作为只读侧车，永远不修改/阻塞数据管道。

### 5.3 双核推理

| 内核 | 本质 | LLM | 职责 |
|------|------|-----|------|
| BIOS 核 | 反射神经 | 永不调用 | DNA 编码 / 32D 校验 / 安全阈值判定 / 五级闸门过滤 |
| Mind 核 | 意识 | 依赖 | 语义理解 / 上下文组装 / 回复生成 / 复杂推理 |

铁律：BIOS 核永不调 LLM，Mind 核永不直接操作存储。

### 5.4 知识库——瑶灵专属图书馆

双模式产品设计：**主线（无感陪伴推送）**——聊天中根据 32D 感知 + 区位信号自动带出匹配场景的知识碎片，零操作负担。**支线（Obsidian 风格 Vault 面板）**——一键唤起，7 组件（文件树/双链/图谱/Dataview/Canvas/编辑器/预览），手动办公。两模式共享同一套 Zvec 向量库 + L0-L3 数据，双向同步。

### 5.5 警幻仙姑运维中枢

8 个批量 API：摘要生成、双链补充、区位标签绑定、画布自动生成、代码注释、表格转换、存量迁移、冷热归档。后台批处理 + 体检工具，不替代前端编辑能力。

## 第七章 · 开发红线

1. 语义层/状态层/知识库逻辑隔离，禁止混表
2. 无 dna_root_id 和 location_fingerprint 的数据拒入库
3. 未经 DNA 编码的记忆数据禁止存储
4. 全局唯一 DNA 编解码器，模块禁建自研解析逻辑
5. DNA 主干生成后永久只读
6. 五级闸门不可关闭，底层硬强制
7. L2 聚类必须叠加区位+时间，禁止纯语义聚类
8. 32D 向量禁止 LLM 直接输出浮点值
9. 32D 永久锁定 32 维，永不扩容/缩维
10. 所有结构体必须经 Transcoder Protobuf 序列化 + CRC32
11. BIOS 核永不调用 LLM，Mind 核永不直接操作存储
12. 单次交互仅生成一颗海胆
13. 产品主线优先无感陪伴知识推送
14. 前端编辑+后台处理底层数据同源同步
15. 禁止 fork Zvec C++ 源码，仅薄封装
16. 跨域指令仅由 Master-Harris 单向发出
