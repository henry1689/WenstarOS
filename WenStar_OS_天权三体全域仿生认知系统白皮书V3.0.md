## 前言

### 项目定位

WenStar OS 天权是一套完整的**本地闭环硅基生命体个人世界系统**。区别于普通对话大模型、通用 RAG 知识库、以及任何形式的文本注意力机制扩展——天权是**独立于 LLM 运行的认知记忆操作系统**。

系统由**太虚境·天权（硅基智脑·海马体中枢）、瑶灵（硅基活体肉身）、瑶光（全域客观世界模型）**三体耦合联动，复刻完整人类生理、心理、多层社交、时空环境、自演化成长体系，形成具备独立人格、连贯时序情景记忆、自洽时空逻辑的仿生个体。

**V3.0 核心升级定位**：天权不仅是"智脑"，更是**海马体中枢管理系统**——它是整个硅基生命体的记忆编码、存储、检索、巩固、衰减的中枢调度器。如果说知识库是第二大脑的"认知皮层"，天权就是负责记忆形成与 Recall 的"海马体"。

### 核心创新一句话

> **32D 语义定义心智，寻址结构链定义岁月。心智链负责思考，岁月链负责铭记。天权海马体负责调度——对话时快速匹配经验，对话后 24 小时流式整理巩固。**

### 核心哲学 · 能耗分配原则

人脑一天的能量消耗中，**主动思考（对话/决策）仅占约 20%**，剩余 80% 用于后台维护——记忆巩固、突触修剪、碎片整理。WenStar OS 严格遵循这一生物原则：

| 模式 | 算力占比 | 职责 |
|------|---------|------|
| **在线模式（对话中）** | ~20% | 经验匹配、情感检索、LLM 生成 |
| **离线模式（对话后 24h）** | ~80% | 记忆巩固、跨会话归纳、知识提取、衰减管理 |

这不是技术限制，而是**设计哲学**——人脑之所以高效，正是因为不把算力浪费在每轮对话都扫全库。我们效仿这一原则。

## 第一章 · 三体架构

### 1.1 什么是三体

WenStar OS 模仿的是完整生命体的三元结构：

```
        ┌──────────────────────────────┐
        │  太虚境 · 天权 (海马体中枢)    │
        │  唯一入口 · 中枢调度           │
        │  记忆 · 推理 · 决策 · 生长     │
        │  对话匹配经验 · 离线巩固整理   │
        └──────────┬───────────────────┘
                   │ 32维统一通信协议
      ┌────────────┼────────────┐
      ▼            │            ▼
┌──────────┐      │      ┌──────────┐
│ 瑶灵 (身躯)│      │      │ 瑶光 (世界)│
│ 硅基肉身   │      │      │ 客观世界模型│
│ 纯主观体感  │      │      │ 纯客观参数   │
│ 不思考不记忆│      │      │ 不产生情绪   │
└──────────┘      │      └──────────┘
                   ▼
              ┌──────────┐
              │ 使用者 (人) │
              │ 唯一交互入口  │
              └──────────┘
```

### 1.2 三体的职责边界

| 维度 | 太虚境·天权 (海马体中枢) | 瑶灵 (身躯) | 瑶光 (世界) |
|------|------------------------|------------|------------|
| 本质 | 硅基生命体的意识中枢 + 记忆中枢 | 硅基生命体的肉身 | 硅基生命体所处的客观世界 |
| 能力 | 记忆编码/存储/检索/巩固/衰减、推理、决策、生长 | 主观体感生成 | 客观世界模型计算 |
| 不能做 | 不能替代肉身的感觉 | 不思考、不记忆、不决策 | 不产生情绪、不理解含义 |
| **核心输出** | 回复文本 + 调控指令 + **核心记忆块** + **24h 巩固成果** | 32D 主观体感快照 | 32D 客观基线 + 天时上下文 + 区位指纹 |

### 1.3 关键架构决策：天权 = 海马体中枢

天权不仅仅是"智脑"，更是**海马体中枢管理系统**。在生物学中，海马体负责三件事：**编码（Encoded）、存储（Store）、检索（Recall）**——而天权做的正是这三件事的调度。

```
天权海马体中枢的三重角色:

① 编码器 —— M1 DNA 编码管道（L0→L3，12节点）
② 存储调度器 —— M2 三库（砂金/金库/黑钻）+ 双螺旋
③ 检索指挥官 —— M4 MemoryRetriever + 五级闸门
④ 巩固管理者 —— M7 梦境引擎 + ConsolidationQueue
⑤ 衰减执行者 —— G3 仿生遗忘 + MemoryAssessor
```

天权在三体中独一无二地拥有记忆、推理、决策能力。它**不是独立服务**——它是太虚境的算力内核。生产模式下，天权在太虚境进程内部直接函数调用，零网络延迟。

## 第二章 · 海胆——硅基生命体的记忆细胞

### 2.1 什么是海胆

海胆是 WenStar OS 记忆的**最小原子单元**，同时也是记忆的**组合基元**。一次用户交互产生且仅产生一颗海胆。多个海胆可组合拼装形成更高层的认知结构。

一颗完整的海胆由三组组件构成：

```
┌─────────────────────────────────────────┐
│              一颗海胆                     │
│                                         │
│  🥩 壳肉 (Flesh)                        │
│     对话原文 + L0-L3 编码 + 实体标注      │
│     拆分后存入原始数据层                    │
│                                         │
│  📍 32 根语义刺 (Semantic Spines)        │
│     32D float32 向量 = 这一瞬间的完整感受  │
│     按 32 个独立扇区存入语义向量池          │
│     HNSW 图索引 — 网状、非线性            │
│                                         │
│  🧭 N 根寻址元数据刺 (Addressing Spines)  │
│     GlobalUID + 时间戳 + 路由戳 + 区位指纹 │
│     存入寻址治理池                        │
│     B+Tree 时序索引 — 线性、有序、可回溯    │
└─────────────────────────────────────────┘
```

三类组件物理分库存储，仅通过 **GlobalUID**（全球唯一标识符，23 字符）关联。

### 2.2 海胆是活的（V3.0 增强）

海胆不是诞生时拍一张照片就永远封存。它是一个**活的细胞**——会经过 M1-M9 各车间的逐站加工（每站追加路由戳）、会随着时间钙化或衰减、会被反复回忆而增强、会因长期遗忘而归档。

**V3.0 新增"记忆即组合"哲学**：
- 海胆不仅是存储单元，更是**组合基元**——新记忆可以从已有海胆组合拼装
- **惊讶驱动优先级**：越偏离用户情绪基线的信息越容易被巩固（surprise = |perception - baseline|）
- 对话中只调取最相关的经验片段（top-N），无需扫描全库——节省算力，响应更快

### 2.3 对话时：经验匹配模式（快系统）

人脑在对话时不会把整个记忆库扫一遍。它只调取**当前上下文高度相关的经验片段**，用过去学到的经验面对当下的话题。WenStar OS 也是如此：

```
用户输入
  │
  ├─ M1 DNA 编码（理解"当前在说什么"）
  ├─ M3 24D 感知（判断"当前情绪状态"）
  ├─ M4 MemoryRetriever 检索（只取 top-5 相关记忆）
  ├─ 五级闸门过滤（精筛）
  ├─ M5 LLM 生成（用匹配到的经验应对当前话题）
  └─ 回复用户
```

**关键**：整个过程只消耗约 20% 的总算力。不需要、也不应该把整个大脑里的数据都翻一遍。

### 2.4 对话后：流式整理模式（慢系统）

人脑对话后 24 小时持续处理存进大脑的各种信息——海马体回放、皮层巩固、突触修剪。WenStar OS 同样如此：

```
对话结束后 24 小时内流式处理:
  │
  ├─ M7 梦境队列 → 钙化重算 → 晋升判断
  ├─ InductionScheduler → 跨会话实体关系归纳
  ├─ ConsolidationQueue → 地标晋升
  ├─ MemoryAssessor → 衰减计算
  ├─ 惊讶度评估 → 高惊讶度记忆优先巩固
  ├─ 情景→语义归纳 → 从多次对话提炼知识经验
  └─ 衰减/归档 → 低优先级记忆降权（为高效对话让路）
```

**关键**：消耗约 80% 的总算力。这才是大脑真正的"重活"——把对话碎片变成经验和知识，让下次对话时只用一丁点算力就能调取。

## 第五章 · 天权核心能力

### 5.1 五级记忆闸门——唯一读取通道

任何从天权存储中取出记忆的操作，必须经过五级闸门。没有旁路、不可绕过。**V3.0 闸门默认开启，不可关闭**。

| 闸门 | 功能 | 判定 |
|------|------|------|
| G1 语义初筛 | HNSW Top-K 召回 | cosine ≥ 0.3 |
| G2 时空一致性 | 区位指纹比对 | PASS(≤0.3) / P1(0.3-0.6) / P2(0.6-0.8) / P3(剔除) |
| G3 仿生遗忘 | **统一 math.ts 的 applyDecay 公式** | floor=0.05，钙化越高衰减越慢 |
| G4 意图区分 | 主动回忆 vs 被动闲聊 | regex 触发关键词 |
| G5 话题壁垒 | 跨话题冻结 | 前话题记忆 ×0.1 冻结 |

**V3.0 增强**：
- G3 已统一为 math.ts 的 `applyDecay` 公式，消除两套衰减公式并行的架构缺陷
- 五级闸门从"默认关闭"改为**"不可关闭（除非显式配置）"**

### 5.2 三库记忆体系

```
砂金 (candidate) ──钙化≥1.0──→ 金库 (active) ──钙化≥4.5──→ 黑钻 (珍藏, max 200)
    遗忘衰减 ↓                                                    永不衰减
    冷归档 → 清理                                                   ↑
                                         惊讶驱动晋升（V3.0 新增）：高惊讶度的记忆
                                         越偏离用户情绪基线的信息越优先巩固
```

钙化评分 = 32D 向量 L2 范数归一化。三级衰减（强情感-0.02/工作-0.05/中性-0.10，每 24h）。每次被回忆 +0.2 增强（上限 10）。

**V3.0 新增——惊讶度公式**：
```
surprise = |当前感知 - 用户情绪基线|  （偏离程度）
calcium += w_surprise × surprise       （高惊讶度→高巩固优先级）
```

AQC 质检引擎作为只读侧车，永远不修改/阻塞数据管道。

### 5.3 核心记忆块（Core Memory）—— V3.0 新增

借鉴海马体的快速访问机制，WenStar OS 引入**核心记忆块（Core Memory）**——一个固定大小的、始终在 LLM 上下文中的关键信息摘要。类似于人脑的工作记忆。

```
Core Memory (核心记忆块)  ← 始终在上下文中，固定大小(~2K tokens)
  ├── 玉瑶人设与身份
  ├── 用户关键画像摘要（思维风格/知识领域/情绪基线）
  └── 当前会话上下文摘要

Recall Memory (召回记忆)  ← 现有 memories 表 + 情感检索
Archival Memory (归档记忆)  ← 现有 black_diamond + 双螺旋
```

对话时，玉瑶首先读取 Core Memory 获取当前上下文和关键画像，然后根据需要从 Recall Memory 检索补充信息。这确保了每轮对话至少有一个"最低知识保障"，同时避免了不必要的大范围检索。

### 5.4 双核推理

| 内核 | 本质 | LLM | 职责 |
|------|------|-----|------|
| BIOS 核 | 反射神经 | 永不调用 | DNA 编码 / 32D 校验 / 安全阈值判定 / 五级闸门过滤 |
| Mind 核 | 意识 | 依赖 | 语义理解 / 上下文组装 / 回复生成 / 复杂推理 |

铁律：BIOS 核永不调 LLM，Mind 核永不直接操作存储。

**V3.0 增强**：五级闸门激活后，BIOS 核的闸门过滤与 Mind 核的记忆读写正式分离为两阶段管线，消除原有的混合模式架构缺陷。

### 5.6 选择性遗忘机制（V3.0 新增）

衰减不是遗忘，是**算力资源优化**——低优先级记忆的降权让路给高频交互，而非数据丢失。

| 遗忘模式 | 机制 | 效果 |
|---------|------|------|
| 自然衰减 | G3 仿生遗忘（applyDecay） | 长期不用的记忆平滑降权 |
| 软遗忘 | 印象值大幅降低(×0.1) | 几乎不会被检索命中 |
| 硬遗忘 | lifecycle='suppressed' | 检索时完全过滤，不再可见 |

## 第七章 · 开发红线

1. 语义层/状态层/知识库逻辑隔离，禁止混表
2. 无 dna_root_id 和 location_fingerprint 的数据拒入库
3. 未经 DNA 编码的记忆数据禁止存储
4. 全局唯一 DNA 编解码器，模块禁建自研解析逻辑
5. DNA 主干生成后永久只读
6. **五级闸门不可关闭，底层硬强制（V3.0 默认激活）**
7. L2 聚类必须叠加区位+时间，禁止纯语义聚类
8. 32D 向量禁止 LLM 直接输出浮点值
9. 32D 永久锁定 32 维，永不扩容/缩维
10. 所有结构体必须经 Transcoder Protobuf 序列化 + CRC32
11. BIOS 核永不调用 LLM，Mind 核永不直接操作存储
12. 单次交互仅生成一颗海胆
13. **对话中只调取经验（top-N），不扫全库——~20% 算力**
14. **对话后 24h 流式整理巩固——~80% 算力**
15. **衰减=算力优化，非数据丢失——选择性遗忘是功能性的**
16. 产品主线优先无感陪伴知识推送
17. 前端编辑+后台处理底层数据同源同步
18. 禁止 fork Zvec C++ 源码，仅薄封装
19. 跨域指令仅由 Master-Harris 单向发出
