## 一、模块间接口契约

### 1.1 M1 DNAEncoder → 各模块

```typescript
// M1 输出的完整 DNA 结构体，各模块统一接收
interface DNAFull {
  trunk: DNATrunk;         // 主干（只读）
  branches: DNABranches;   // 分支挂载区
}

interface DNATrunk {
  globalUID: string;           // 全局唯一 ID（格式见 §1.3）
  version: number;             // 规范版本号
  createdAt: number;           // Unix 秒
  checksum: string;            // CRC32
}

interface DNABranches {
  semantic?: SemanticBranch;   // 语义链（32D 向量）
  address?: AddressBranch;     // 寻址链（复合管控）
  lifecycle?: LifecycleBranch; // 生命周期
  entity?: EntityBranch;       // 实体归属
}
```

### 1.2 GlobalUID 生成规范

```
格式: TTNNNNBBBLLLLLLSSSSSS
       TT     = 类型标记 (2 char): "MM"=记忆 "ZK"=知识库 "PR"=画像
       NNNN   = 节点编号 (4 char): 机器标识
       BBB    = 批次号 (3 char): 会话批次
       LLLLLL = 区位标识 (6 char): 瑶光输出
       SSSSSS = 随机盐 (6 char): crypto.randomBytes(3).toString('hex')

示例: "MM0001A3B2F1A0C4D5E6F7"
```

### 1.3 语义链 32D 扇区分配

| 扇区 | 维度 | 名称 | 来源 | 值域 | 说明 |
|------|------|------|------|------|------|
| 00-05 | 6D | 外源情绪谱 | 瑶光六维拮抗式 | [-1.0, 1.0] | 对方语气/表情/氛围 |
| 06-10 | 5D | 生理躯体 | 瑶灵感知 | [-1.0, 1.0] | 疼痛/疲劳/能量/激素 |
| 11-16 | 6D | 内在精神 | LLM 辅助+模板 | [-1.0, 1.0] | 自我/欲望/共情/自保 |
| 17-22 | 6D | 圈层人际 | LLM 辅助+模板 | [-1.0, 1.0] | 家庭/亲友/职场/依恋 |
| 23-28 | 6D | 时空环境 | 瑶光直出 | [-1.0, 1.0] | 居所/职场/户外/区位/气象 |
| 29-31 | 3D | 生长引擎 | 继承+D32 统筹 | [-1.0, 1.0] | 探索/求知/物理成长 |

## 二、跨模块通信协议

### 2.1 内部模块调用

- 格式：TypeScript 接口同步调用
- 约束：**必须传递完整 DNA 结构体**，禁止仅传 `globalUID` 片段
- 异常处理：调用方捕获异常 → 写入 error log → 走降级路径（详见《逻辑链设计》§5）

### 2.2 HTTP 对外接口

```typescript
// 请求体格式
interface TianquanRequest {
  dna: DNAFull;                        // 完整 DNA
  query: string;                       // 用户输入
  mode: 'semantic_only' | 'dual_chain'; // 检索模式
}

// 响应体格式
interface TianquanResponse {
  dna: DNAFull;                        // 更新后的 DNA
  reply: string;                       // 生成回复
  memories_accessed: string[];         // 被访问的记忆 globalUID 列表
  gate_log: GateLog;                   // 五级闸门过滤日志
}
```

### 2.3 数据库表

- 所有核心表必须包含 `global_uid` 字段作为双链关联键
- 语义表与寻址表之间通过 `global_uid` 做 JOIN，禁止自建关联逻辑

## 三、数据库 Schema 变更管控

1. 所有 DDL 变更必须走架构评审
2. 新增字段仅允许 `ALTER TABLE ... ADD COLUMN`（追加模式）
3. 禁止修改已有字段类型、禁止删除字段、禁止重命名
4. 每次 Schema 变更记录版本号

## 四、禁止操作清单

| # | 禁止操作 | 检测方式 |
|---|---------|---------|
| 1 | 各模块自建 GlobalUID 生成逻辑 | grep `generateUID` 非 `DNAEncoder.ts` 文件 |
| 2 | 绕过全局 DNA 编解码器直接读取 raw bytes | grep `Buffer.from` 在 `Transcoder.ts` 外使用 |
| 3 | 时序字段出现在语义向量构造代码中 | grep `timestamp\|unix` 在 `PerceptionAnalyzer.ts` 中 |
| 4 | 语义向量维度超过 32 | lint rule: `assert(semanticVector.length === 32)` |
| 5 | 写入数据未经过 `DNA.validate()` | pre-commit hook |
| 6 | 硬编码 DNA 字段（应使用 proto 定义）| grep 硬编码偏移量 |
