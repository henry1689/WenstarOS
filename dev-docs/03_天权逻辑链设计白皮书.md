           ▼
┌─────────────────────────────────────────────────────┐

│   失败: 单路异常→继承；双路异常→FATAL 阻断             │

## 二、状态机定义

### 2.1 BIOS 层状态机

```
              ┌─────────┐
    启动 ───→ │  INIT    │ → 加载配置/DB 连接
              └────┬────┘
                   ↓
              ┌─────────┐
              │  IDLE    │ ← 等待输入
              └────┬────┘
                   ↓ 收到双路 32D
              ┌─────────┐
              │ MERGING  │ → 逐维度比对融合
              └────┬────┘
           ┌───────┼───────┐
           ↓       ↓       ↓
      正常融合   异常降级   FATAL
      → IDLE    → IDLE   → HALT
                          (需人工介入)
```

### 2.2 检索模式状态

```
              ┌─────────────┐
              │ SEMANTIC     │ ← 默认：纯语义链
              │ _ONLY        │   寻址链完全挂起
              └──────┬──────┘
                     │ 检测到 "昨天/上次/回忆/记得"
                     ↓
              ┌─────────────┐
              │ DUAL_CHAIN   │ ← 寻址链定界 + 语义精排
              └──────┬──────┘
                     │ 回复完成
                     ↓
              ┌─────────────┐
              │ SEMANTIC     │ ← 恢复默认
              │ _ONLY        │
              └─────────────┘
```

### 2.3 记忆生命周期状态

```
WRITE → HOT(热) → COLD(冷) → ARCHIVE(归档)
          ↑           │
          └───────────┘
          被主动回溯时升温
```

## 三、五级闸门决策树

```
候选记忆集 entry
  │
  ├─ G1 语义初筛: keyword match OR vector cosine > 0.65?
  │    NO  → DISCARD
  │    YES → continue
  │
  ├─ G2 时空校验:
  │    distance = cosine(current_loc, entry_loc)
  │    ├─ ≤ 0.3 → PASS, continue
  │    ├─ ≤ 0.6 → P1: text → summary_only
  │    ├─ ≤ 0.8 → P2: weight × 0.3, exit_main_queue
  │    └─ > 0.8 → P3: DISCARD
  │
  ├─ G3 遗忘衰减:
  │    decayed_weight = original × e^(-λ × elapsed_hours)
  │    if env_changed: λ ×= 3
  │    if decayed < 0.05: DISCARD
  │
  ├─ G4 意图区分:
  │    if intent == 'passive_chat' AND entry.level != 'L2' AND entry.level != 'L3':
  │      DISCARD (闲聊不加载 L0/L1)
  │    if intent == 'active_recall': ALLOW_ALL
  │
  └─ G5 话题壁垒:
       topic_shift = cosine(current_topic_vec, entry.topic_vec)
       if topic_shift > 0.7: FREEZE entry (暂存,不做永久丢弃)
       else: ALLOW
```

## 四、异常路径与降级策略

| 故障点 | 降级行为 | 恢复 |
|--------|---------|------|
| 瑶光 外源谱获取失败 | 用中性值 [0,0,0,0,0,0] 替代 | 瑶光重连后自动恢复 |
| 瑶灵 躯体感知异常 | 继承上一轮 D06-D10 值 | 下次正常采样覆盖 |
| BIOS 双路融合一方异常 | 正常方的值 + `inherited` 标记 | 下次融合覆盖 |
| DNA validate() 失败 | FATAL: 阻断写入 + 告警 | 检查畸形原因→修复→重试 |
| 记忆检索超时 (>500ms) | 返回空候选集，纯 LLM 回答 | 下轮恢复 |
| 知识库检索失败 | 仅用记忆候选 | ZvecEngine reconnect |
| 寻址链损毁(部分 GlobalUID 丢失) | **降级为纯语义模式** | 黑匣子谱系台账反向还原 |
| 语义链比特漂移 | 寻址链 CRC 检测→隔离标记→从快照恢复 | 恢复后解除隔离 |
| LLM API 超时 | 返回预设兜底回复 | 重试 3 次 |

## 五、双检索模式切换逻辑

```typescript
function selectMode(query: string, dna: DNAFull): 'semantic_only' | 'dual_chain' {
  // 触发词检测
  const timeTriggers = /昨天|上周|去年|记得|回忆|复盘|追溯|那年|当时/;
  const personTriggers = /他|她|爸爸|妈妈|同事|那个.*人/;
  const locationTriggers = /在.*的时候|在.*地方|办公室|家[里里]|浴室/;

  if (timeTriggers.test(query) || 
      personTriggers.test(query) || 
      locationTriggers.test(query)) {
    return 'dual_chain';
  }
  return 'semantic_only';  // 默认
}
```

## 六、并发与竞态控制

- SQLite WAL 模式：允许多读单写
- DNA 写入串行队列：保证 `global_time_seq` 严格递增
- 读取快照隔离：每次检索使用当前时刻的一致性快照
- 异步任务(L1/L2/L3)使用独立 worker 进程
