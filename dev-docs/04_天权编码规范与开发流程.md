## 一、TypeScript 编码标准

### 1.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | kebab-case | `five-stage-gate.ts` |
| 类/接口 | PascalCase | `class FiveStageGate` / `interface DNABranches` |
| 函数/变量 | camelCase | `computeDecayedWeight()` / `globalUID` |
| 常量 | UPPER_SNAKE | `LOCATION_THRESHOLD_A = 0.3` |
| 私有成员 | `_` 前缀 | `private _hotCache: Map` |
| DNA 字段 | snake_case | `global_uid`, `time_slice_tag` |
| SQL 表名/列名 | snake_case | `state_spines`, `atom_address_timeline` |
| Proto 字段 | snake_case | `dna_id`, `dimension_id` |

### 1.3 错误处理

```typescript
// 三级错误
// FATAL: 阻断 + 告警 + 不进下一环节
throw new TianquanFatalError('DNA主干校验失败', { globalUID });

// ERROR: 告警 + 丢弃当前数据 + 继续运行
logger.error('分支编码错位', { branch: 'entity', globalUID });
return null;  // 不参与后续运算

// WARN: 告警 + 降级兼容 + 继续运行
logger.warn('DNA版本不匹配，使用默认兼容解析', { version: 2, expected: 1 });
```

### 1.4 函数签名规范

```typescript
// ✅ 正确：DNA 完整传递
function retrieveMemory(dna: DNAFull, query: QueryParams): Promise<MemoryAtom[]>

// ❌ 错误：仅传片段
function retrieveMemory(globalUID: string, query: QueryParams): Promise<MemoryAtom[]>

// ✅ 正确：异常路径清晰
function safeDecode<T>(decodeFn: (buf: Buffer) => T, buf: Buffer): T | null

// ❌ 错误：吞异常
function decode(buf: Buffer): any { try { ... } catch {} }
```

## 二、Git 分支策略

```
main ────────────────────────────── (保护分支，仅 PR 合并)
  │
  ├── feature/p0-five-stage-gate
  ├── feature/p1-l0-l3-distillation
  ├── feature/p2-zvec-engine
  ├── feature/p3-32d-perception
  ├── feature/p4-protobuf-transcoder
  └── fix/demo-*
```

**规则**:
- `main` 分支保护：禁止直接 push，必须 PR + 1 人 review
- feature 分支命名：`feature/p{n}-{module-name}`
- 每次 commit 前必须通过 lint + test
- 合并前必须通过 CI（类型检查 + 单元测试 + lint）

## 三、Code Review 清单

### 必须检查

- [ ] 是否绕过全局 DNA 编解码器？
- [ ] 是否在语义向量构造中包含时序字段？
- [ ] 是否硬编码了维度数量（应使用常量）？
- [ ] 函数是否接收完整 DNA 结构体而非片段？
- [ ] 存储写入前是否调用 `DNA.validate()`？
- [ ] 新增字段是否在 proto 中有对应定义？
- [ ] 异常是否正确分类（FATAL / ERROR / WARN）？
- [ ] 是否为纯函数增加单元测试？

## 四、Proto 文件规范

```bash
# 编译命令
npx pbjs -t static-module -w commonjs \
  -o src/transcoder/proto.js \
  spine.proto token.proto zvec_entry.proto address_timeline.proto

npx pbts -o src/transcoder/proto.d.ts src/transcoder/proto.js
```

**规则**:
- `.proto` 源文件放在 `src/transcoder/proto/`
- 编译产物不提交 Git，在 CI 中生成
- 字段编号永久保留，删除字段只标记 `reserved`
- 新增字段使用新编号（跳号也接受）

## 五、SQL 建表规范

- 表名/列名: snake_case
- 主键: `PRIMARY KEY (col1, col2)` 或 `WITHOUT ROWID`（复合主键）
- 索引: `CREATE INDEX IF NOT EXISTS idx_{table}_{col}`
- 约束: `CHECK` 用于值域校验
- 所有表包含 `created_at INTEGER NOT NULL DEFAULT (unixepoch())`
- 二进制数据使用 `BLOB` 类型

## 六、禁止事项自动化检查

```bash
# pre-commit hook 示例检查
#!/bin/sh

# 检查是否有绕过 Transcoder 的裸 JSON 写入
if git diff --cached --name-only | grep '\.ts$' | xargs grep -l 'JSON.stringify.*memory' 2>/dev/null; then
  echo "禁止裸 JSON 写入记忆，请使用 Transcoder.encode()"
  exit 1
fi

# 检查语义向量维度
if git diff --cached --name-only | grep 'PerceptionAnalyzer' | xargs grep -l '\.length.*33\|33.*dimension' 2>/dev/null; then
  echo "语义向量维度固定为 32，禁止扩容"
  exit 1
fi

# 检查是否有硬编码的 DNA 偏移量
if git diff --cached --name-only | grep '\.ts$' | xargs grep -l 'substring.*\d+.*\d+' 2>/dev/null; then
  echo "禁止硬编码 DNA 字段偏移量，请使用 DNA.decodeBranch()"
  exit 1
fi
```
