## 二、WAL 崩溃恢复流程

```
系统启动
  ↓
检查 WAL 文件完整性
  ├── 完整 → 重放 WAL → 恢复内存索引 → 正常启动
  └── 损坏 → 从最近快照恢复 → 重放 WAL 剩余段 → 正常启动
       └── 两者都损坏 → 警告：记忆可能有损 → 从寻址链反向校验语义链

恢复优先级:
1. 寻址谱系(小体积) → 分钟级恢复
2. 语义链索引(HNSW) → 从持久化段重建
3. LRU 缓存 → 丢弃(冷重启)
```

## 三、双链单侧故障降级

| 故障 | 行为 | 恢复 |
|------|------|------|
| **语义链索引损坏** | 查询全部降级为寻址链过滤 + 原文关键词匹配 | 从语义快照重建 HNSW |
| **寻址链 B+树损坏** | 自动切换纯语义模式，所有查询走 HNSW | 从谱系台账反向重建 |
| **语义链个别扇区损坏** | 该扇区值用中性值替代，标记 `isolated` | 等待下轮正常写入覆盖 |
| **寻址链个别 GlobalUID 丢失** | 从语义链反向查找，重建归属关系 | 自动修复 |

## 四、冷热数据自动迁移

```typescript
// 每日凌晨 Cron
async function coldHotMigration(): Promise<void> {
  const now = Date.now();
  const THIRTY_DAYS = 30 * 86400 * 1000;
  const TWO_YEARS = 2 * 365 * 86400 * 1000;

  // 温→冷 (30 天没被访问)
  await db.run(`
    UPDATE atom_address_timeline
    SET hot_cold_level = 'C'
    WHERE absolute_timestamp < ? AND hot_cold_level = 'W'
  `, [now - THIRTY_DAYS]);

  // 冷→归档 (2 年以上)
  await db.run(`
    UPDATE atom_address_timeline
    SET hot_cold_level = 'A'
    WHERE absolute_timestamp < ? AND hot_cold_level = 'C'
  `, [now - TWO_YEARS]);

  // 归档数据的语义链降权（衰减系数 × 10）
  await db.run(`
    UPDATE state_spines
    SET consistency_mark = 'overridden'
    WHERE dna_id IN (
      SELECT global_uid FROM atom_address_timeline WHERE hot_cold_level = 'A'
    )
  `);
}
```

## 五、备份体系

| 数据 | 方式 | 频率 | 保留 |
|------|------|------|------|
| 语义本体（state_spines） | 全量快照 (SQLite .backup) | 每日 1 次 | 最近 7 天 |
| 寻址谱系（atom_address_timeline） | **增量备份** (WAL 变更文件) | **每分钟** 1 次 | 最近 30 天 |
| ZVEC 知识库 | 全量 SQLite .backup | 每日 1 次 | 最近 7 天 |
| DNA 配置/规则 | Git 版本控制 | 每次变更 | 永久 |

**为什么寻址谱系用增量？**
- 寻址链体积远小于语义链（每条 ~200 bytes vs ~5KB）
- 分钟级增量保证**时序不会断裂**——即使语义本体丢了，寻址谱系还能告诉系统「发生了什么」

## 六、磁盘空间监控与自动 GC

```typescript
// 每小时
async function diskMonitor(): Promise<void> {
  const freeGB = getDiskFree('/data/tianquan');
  
  if (freeGB < 10) {
    logger.warn(`磁盘空间不足: ${freeGB}GB`);
    
    // 自动清理过期归档
    await db.run(`DELETE FROM state_spines WHERE dna_id IN (
      SELECT global_uid FROM atom_address_timeline 
      WHERE hot_cold_level = 'A' AND absolute_timestamp < ?
    )`, [Date.now() - 3 * 365 * 86400 * 1000]);
    
    // 强制 compact WAL
    await db.run('PRAGMA wal_checkpoint(TRUNCATE)');
  }
  
  if (freeGB < 2) {
    logger.fatal(`磁盘空间严重不足: ${freeGB}GB, 进入只读模式`);
    // 拒绝所有新写入，保持读取
  }
}
```
