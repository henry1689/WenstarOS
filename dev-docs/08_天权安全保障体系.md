# 天权安全保障体系 V1.0

**适用**: 全栈 / 安全审计  
**生效阶段**: 全局适用（P0 启动时基础安全措施同步上线）  

---

## 一、数据加密

### 1.1 静态数据加密

| 数据 | 加密方式 | 密钥管理 |
|------|---------|---------|
| SQLite 数据库文件 | SQLCipher (AES-256-GCM) | 用户主密钥 → PBKDF2 派生 |
| Protobuf BLOB 字段 | 不额外加密（SQLCipher 已覆盖） | — |
| 备份文件 | GPG 对称加密 | 备份专用密钥 |

### 1.2 传输加密

| 链路 | 加密 | 说明 |
|------|------|------|
| 瑶灵→太虚 (内部) | 不加密（进程内调用） | 本地单机 |
| 瑶光→太虚 (内部) | 不加密（进程内调用） | 本地单机 |
| WebUI → 后端 HTTP | **HTTPS (TLS 1.3)** | 自签证书（本地）/ Let's Encrypt（公网） |
| DeepSeek API 调用 | HTTPS | API Key 在环境变量中 |
| 手机公网模式 | Cloudflare Tunnel + HTTPS | 无需开放端口 |

### 1.3 密钥管理

```
用户主密钥（本地生成，永不上传）
  ↓ PBKDF2(salt, iterations=600000)
派生密钥 → SQLCipher 打开数据库
  ↓
环境变量(.env.local) → 加密存储 API Key
  ↓ Git
.gitignore 排除 .env.local（禁止提交密钥）
```

## 二、访问控制

### 2.1 当前模式：本地单用户

- 天权运行在用户本机
- 数据不经过任何远程服务器
- 无需认证层（进程内访问）

### 2.2 未来扩展：多用户隔离

当需要多角色/多用户独立记忆池时：

```
/user_data/{user_id}/
  ├── tianquan.db        # 独立 SQLite 实例
  ├── address_timeline/   # 独立寻址链
  └── zvec_data/          # 独立向量索引
```

每个用户拥有完全物理隔离的记忆数据库。通过 `user_id` + `globalUID` 前缀区分。

## 三、隐私保护

### 3.1 核心原则

| 原则 | 实现 |
|------|------|
| **数据不上云** | 全部记忆、知识库本地 SQLite 存储 |
| **LLM API 最小化** | 只发送 L2/L3 轻量化摘要，不发送原始对话全文 |
| **DNA 脱敏** | 对外接口不暴露 raw DNA 结构体 |
| **用户可控删除** | 提供「遗忘某段记忆」功能，双链同步删除 |

### 3.2 用户遗忘权实现

```typescript
async function forgetMemory(globalUID: string): Promise<void> {
  // 1. 标记寻址链为已删除
  await db.run(
    `UPDATE atom_address_timeline SET state_flag = 'F' WHERE global_uid = ?`,
    [globalUID]
  );
  // 2. 语义链标记为已删除
  await db.run(
    `UPDATE state_spines SET consistency_mark = 'overridden' WHERE dna_id = ?`,
    [globalUID]
  );
  // 3. 异步从 HNSW 索引中移除（延迟到下次 compaction）
  await hnswIndex.markDeleted(globalUID);
  // 4. 记录操作日志
  await auditLog('forget', globalUID);
}
```

## 四、审计日志

### 4.1 日志记录

```sql
CREATE TABLE IF NOT EXISTS audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    action TEXT NOT NULL,          -- 'read'/'write'/'delete'/'forget'/'encode'/'decode'
    global_uid TEXT,
    module TEXT NOT NULL,          -- 'm1'/'m2'/'m4'/'transcoder'/'zvec'
    result TEXT NOT NULL,          -- 'ok'/'error'/'fatal'/'degraded'
    detail TEXT,                   -- JSON
    created_at INTEGER NOT NULL DEFAULT (unixepoch())
);
```

### 4.2 日志保留

- 在线日志：保留 90 天
- 归档日志：保留 1 年（压缩存储）
- 安全事件日志：永久保留

## 五、供应链安全

| 风险 | 缓解 |
|------|------|
| @zvec/zvec 依赖投毒 | `package-lock.json` + `npm audit` 每次 CI |
| @zvec/zvec 版本锁定 | 固定版本号，禁止 `^` 范围 |
| Protobuf 编译产物篡改 | `.proto` 为源，`.js`/`.d.ts` 为 CI 生成，不提交 Git |
| LLM API Key 泄露 | `.env.local` + `.gitignore`，pre-commit hook 检测 |
| npm 依赖漏洞 | `npm audit --production` 每次 PR；高危立即修复 |

## 六、安全红线

| # | 红线 | 违反后果 |
|---|------|---------|
| 1 | **禁止明文传输 DNA 结构体**（HTTP 外网） | 立即阻断 |
| 2 | **禁止未校验数据入库**（必须经 `DNA.validate()`） | 写入层拒绝 |
| 3 | **禁止提交密钥/Token/密码到 Git** | pre-commit hook 阻断 |
| 4 | **禁止绕过 Transcoder 裸写数据库** | Code Review 一票否决 |
| 5 | **禁止向 LLM API 发送完整原始记忆**（仅发送 L2/L3 摘要） | 架构级违规 |
| 6 | **禁止直接暴露文件系统路径给外部 API** | 安全评审 |

## 七、安全事件响应

| 事件 | 响应 |
|------|------|
| DNA 畸形检测（审计扫描发现） | 自动隔离 → 通知管理员 → 从快照恢复 |
| 磁盘异常写入 | 切换只读模式 → 保留现场 → 分析原因 |
| API Key 泄露 | 立即轮换密钥 → 检查审计日志是否有异常访问 |
| SQLite 文件被外部篡改 | SQLCipher MAC 校验失败 → 数据库拒绝打开 → 从加密备份恢复 |
