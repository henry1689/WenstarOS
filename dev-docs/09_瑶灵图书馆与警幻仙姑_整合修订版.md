| 维度 | 主线（无感陪伴） | 支线（办公窗口） |
|------|---------------|----------------|
| 用户画像 | 大众 / 陪伴向 | 办公 / 研发 / 专业人群 |
| 优先级 | **最高**（产品核心壁垒） | 辅助线（满足刚需） |

## 二、白皮书增补修订

### §7 瑶灵专属图书馆（ZVEC 第二大脑体系）

#### 7.1 产品双模式定位

**模式一：原生仿生陪伴模式（主推核心体验）**

对标所有 AI 产品的核心差异化：
- 无需手动打开文档面板
- 无需手动分类打标
- 无需主动检索文件
- 依托瑶灵 36D 躯体感知 + 瑶光时空区位信号
- 在日常闲聊、独处、陪伴中**潜移默化推送**匹配当下场景/心境的碎片化知识点
- 像伴侣闲谈、像导师顺势点拨——无刻意任务感
- 上传素材后自动分层、自动绑定区位、自动计算知识关联——全程零维护

**模式二：轻量化 Obsidian 风格办公交互模式（专业辅助支线）**

内置可手动唤起的独立 Vault 可视化面板，完整复用 Obsidian 成熟高效办公交互逻辑：
- 标准文件树目录 + 多标签 MD 编辑窗口
- `[[内部双链]]` 双向关联 + 图谱可视化面板
- Dataview 标签/时序 SQL 筛选 + 自定义笔记模板
- Canvas 拖拽画布 + 思维导图可视化
- 批量导入/导出/标签/摘要/重命名 自动化工具
- 多模态预览（表格/代码/图片/音视频内嵌）

**互通规则**：两种模式共用同一套本地 Vault 文件、同一 Zvec 向量库、同一套 L0-L3 分层数据。窗口手动编辑/新增的内容**实时同步**至瑶灵对话自动推送管线。双向数据完全打通，不存在两套隔离知识库。

#### 7.2 底层载体与引擎不变

- **存储载体**: 本地独立 Vault 文件夹，标准 MD 格式，文件夹复制即可全量备份迁移
- **向量底层**: 阿里 Zvec，稠密语义 + 稀疏标签 + 区位标量混合检索，统一送入五级时空闸门
- **全自动后台管线**: 文件上传自动 L0-L3 分层、自动区位指纹绑定、自动相似度双链生成

#### 7.3 双模式联动运行逻辑

1. 日常默认关闭 Vault 窗口，走无感陪伴主线
2. 用户需要办公/批量整理/精细化规划时，一键唤起 Obsidian 式交互面板
3. 窗口内所有编辑、新增、标签、双链修改实时同步底层 Zvec 元数据
4. 警幻仙姑同时支撑两条管线：日常自动巡检修复 + 窗口批量办公自动化

### §8 警幻仙姑 · 全域系统运维中枢

#### 8.1 原有核心职能（保留）

1. 全域双库体检修复（36D 记忆 + ZVEC 图书馆）
2. 隐私安全、异常数据隔离防护
3. 基础文档批量整理
4. 系统使用指引问答

#### 8.2 新增：办公自动化专项能力

| 能力 | 说明 |
|------|------|
| `batch_summary` | 批量 MD 摘要生成 |
| `batch_auto_link` | 批量标签/双链自动补充 |
| `batch_tag_scene` | 多文件时序聚类、批量区位标签绑定 |
| `canvas_auto_build` | Canvas 画布自动生成 |
| `batch_code_comment` | 批量代码注释 |
| `table_convert` | 表格结构化转换 |
| `vault_migrate` | 存量文档一键迁移、格式标准化 |
| `vault_archive` | 定时知识库归档、冷热素材自动分区 |

#### 8.3 边界约束

警幻仅为后台批量处理、体检工具。**不替代**前端 Vault 交互窗口的手动编辑能力。

## 三、工程蓝皮书增补章节

### §19 ZVEC 瑶灵图书馆工程实现规范

#### 19.1 双模式架构隔离但数据同源

```
自动推送管线（M4 内置，无前端强依赖）
  ├── 无 UI 面板
  ├── 对话流程自动调用 Zvec 检索
  └── 输出: 匹配当下场景的知识碎片 → LLM Prompt

可视化 Vault 前端模块（可选唤起独立窗口）
  ├── 轻量化复刻 Obsidian 交互组件
  ├── 仅作为上层操作入口
  └── 所有操作实时写入底层 Zvec 元数据

底层统一（二者共享）:
  ├── Vault 文件目录（标准 MD）
  ├── Zvec 多 Collection（稠密 + 稀疏）
  ├── L0-L3 分层表
  └── 双链元数据表
```

#### 19.2 Vault 前端面板开发标准

**必须实现的组件**:

| 组件 | 说明 | 对标 Obsidian |
|------|------|-------------|
| FileTree | 树形文件浏览器 | 左侧边栏 |
| TabEditor | 多 Tab MD 编辑器 | 主编辑区 |
| WikiLink | `[[双链]]` 解析 + 双向关联 | 核心功能 |
| GraphView | 知识图谱可视化面板 | 关系图谱 |
| Dataview | 标签/时间筛选查询 | 数据检索 |
| Canvas | 拖拽画布 + 思维导图 | 无限画布 |
| Preview | 多模态预览（表格/代码/图片/音视频） | 预览面板 |

**限制约束**:
- 不做重度第三方插件生态
- 仅内置原生办公自动化基础功能
- 复杂批量操作统一交由警幻后台接口执行

#### 19.3 双向数据同步硬性规则

| # | 规则 | 触发时机 |
|---|------|---------|
| 1 | 前端 Vault 窗口手动新增/修改 → `实时写入` Zvec 元数据表 → `触发向量增量更新` | onSave |
| 2 | 后台自动生成双链/L1/L2 摘要/区位标签 → `同步渲染`至 Vault 前端图谱/文件标签面板 | onComplete |
| 3 | 关闭窗口后 `自动保存` 所有改动 | onClose |
| 4 | 禁止存在两套独立文件/向量存储 | 架构级约束 |

#### 19.4 异步管线适配双模式

文件上传分两条异步分支：
- **轻量即时分支**: 供 Vault 窗口实时预览（MD 渲染 → 即时可见）
- **完整后台分支**: 分层蒸馏 → 向量化 → 区位绑定 → 自动双链生成 → 同步服务对话推送管线

### §20 警幻仙姑运维模块（新增办公批量接口）

```typescript
// 原有接口保留
interface JingHuanCore {
  healthCheck(): HealthReport;

  batchGenerateSummary(fileIds: string[]): Promise<SummaryResult[]>;     // 批量 L2 摘要

}
```
