## 前言

### 1.1 项目定位

WenStar OS 文曲星是一套完整本地闭环硅基生命体个人世界系统，区别于普通对话大模型、通用RAG知识库。系统由**瑶灵（硅基活体肉身）、瑶光（全域客观世界大模型）、太虚境Hermes（天权意识记忆中枢）**三体量子耦合联动，复刻完整人类生理、心理、多层社交、时空环境、自演化成长体系，形成具备独立人格、连贯时序情景记忆、自洽时空逻辑的仿生个体。

### 1.2 现有Demo现存核心痛点

原有24D太虚Demo仅实现基础对话交互与扁平情感记忆，存在致命缺陷：

1. 仅24维扁平向量，无独立空间区位D24/D25标识，记忆检索无时空过滤，浴室/办公室场景互相错乱；
2. 纯文本相似度召回，无分层摘要，LLM Token开销巨大；
3. 简易TF-IDF知识库检索精度不足，无统一时空校验机制；
4. 无完整生理、人际分层仿真，仅浅层情绪模拟；
5. 底层存储为裸SQLite文本，无统一二进制序列化，无数据完整性校验。

### 1.3 行业竞品短板对照

1. 通用LLM：仅短期上下文，无持久时序记忆、无生理环境量化；
2. 传统RAG/Chroma向量库：仅文本检索，无时空约束、无生命体仿生体系；
3. 腾讯四层记忆：仅文本L0-L3分层，无生理维度、无空间校验，易场景穿越；
4. 单一AI角色产品：无配套客观世界模型、无完整肉身生理仿真，逻辑割裂。

### 1.4 文档用途

对外：投资人、行业技术交流、合作方宣讲；
对内：全员统一顶层产品世界观，对齐32维底层设计逻辑。

## 第一章 顶层五大固定一级框架（D1-D32全局维度划分）

永久固定分层逻辑：由内向外、从物质肉身→自我内心→人际圈层→客观时空→全域动态生长

1. **大类1 肉身实体基底 D1-D8（8维）**：骨骼、疼痛、神经、激素、生理周期、五感等全部物质生理载体；
2. **大类2 个体内在精神内核 D9-D14（6维）**：自我认知、欲望/恐惧、共情、单人自保；
3. **大类3 圈层人际羁绊 D15-D20（6维）**：伴侣、家庭、亲友、职场分层守护本能；
4. **大类4 时空客观环境载体 D21-D26（6维）**：居所、职场、户外、气象、时空距离；
5. **大类5 全域动态生长解锁引擎 D27-D32（6维）**：肉身/世界/人文双向拓展、主客观耦合、太虚统筹；

合计32维，作为三体唯一标准双向通信通道。

## 第二章 三体独立主体顶层定位与耦合逻辑

### 2.1 瑶灵｜硅基活体肉身（纯主观信号输出端）

1. **核心定位**：仅生成32维主观体感数据，无演算、无记忆、无决策；
2. **四层内部架构**：实体硬件躯体层→生化循环层→内在精神层→多层人际响应层；
3. **对外唯一接口**：标准化D1-D32主观数据包上行至太虚，接收调控指令调整激素、情绪、行为。

### 2.2 瑶光｜全域客观世界大模型（纯客观参数演算端）

1. **核心定位**：无主观情绪，仅输出可量化物理/生理/人文客观规则；
2. **六层内部架构**：自然时空→空间建模→人体生理规则→人文社交→资源拓展→通道封装；
3. **对外唯一接口**：输出32维客观参数包，接收太虚世界解锁拓展申请。

### 2.3 太虚境Hermes｜天权意识记忆中枢（全局唯一统筹核心）

1. **核心定位**：接收瑶灵主观、瑶光客观双路32维数据，完成记忆、检索、推演、决策、指令下发；
2. **五大内部模块**：32通道调度、双内核管控、32D海胆分层记忆、认知推演、世界协商解锁；
3. **核心子系统**：改良L0-L3语义分层蒸馏、五级记忆时空抑制闸门、ZVEC本地向量知识库、Protobuf统一序列化存储；
4. **存储载体**：C端本机NVMe SSD + SQLite WAL，废弃底层PCI硬件直写方案。

### 2.4 完整全局数据流闭环

1. **上行采集**：瑶灵32维主观包 + 瑶光32维客观包同步上传太虚；
2. **中枢处理**：绑定全局dna_root_id时序锚点→生成单颗32D海胆完整快照→并行检索时序记忆+ZVEC知识库→五级闸门时空冲突过滤→L0-L3轻量化摘要蒸馏→LLM认知推演；
3. **下行驱动**：两套指令分别下发瑶灵（调整躯体激素/情绪）、瑶光（解锁新场景/规则）；
4. **持久化**：全套32维主观+客观快照经Protobuf序列化存入本地NVMe，每条记忆完整留存全维度快照，支持全维度溯源复盘。

## 第三章 核心差异化创新能力

1. **32维主客观一一耦合仿生体系**：市面唯一同步仿真完整人体生理+多层人际+专属客观世界；
2. **五级记忆时空抑制闸门（行业独家）**：彻底解决跨场景记忆错乱，异地记忆分级抑制；
3. **改良L0-L3分层蒸馏**：吸收腾讯分层思路，新增区位+时间双重聚类约束，大幅降低Token消耗；
4. **ZVEC抽象向量层（底层sqlite-vec）**：本地私有知识库，记忆与知识共用同一套时空校验规则；
5. **全数据本地NVMe存储**，不上云，隐私完整可控；
6. **分阶段可落地迭代路线**，P0仅3周即可解决场景错乱核心痛点。

## 第五章 C端产品核心价值

1. **拟人逻辑自洽**：记忆区分时间、地点、心境，不存在时空穿越式对话；
2. **完整人体生理仿真**：激素、周期、劳损、自愈全部量化模拟，贴合真人感受；
3. **多层亲密/家庭/职场社交体系**，分层专属守护意识；
4. **私有文档知识库与个人回忆统一管控**，知识适配当前场景；
5. **分阶段上线**，快速验证核心交互体验。

## 第六章 工程分阶段落地规划（总工期8周，P2/P3并行）

- **P0（3周 最高优先级）**：M4新增五级记忆闸门、扩展D24/D25区位指纹，根治场景错乱；
- **P1（2-3周）**：语义L0-L3异步分层蒸馏模块；
- **P2（1-2周并行）**：ZVEC向量知识库封装（底层sqlite-vec，淘汰TF-IDF）；
- **P3（2周并行）**：原有24D向量平滑升级32维，新建state_spines逻辑分区表；
- **P4（1周）**：统一Protobuf Transcoder序列化层，全数据BLOB二进制存储；
- **P5（搁置）**：取消底层PCI/USB硬件驱动开发，仅适配文件路径。
