## 序言 · 闸门为什么是五级

AI 记忆系统最常见的失败模式是"场景错乱"——在办公室聊天时错误地回忆起浴室的对话。这不是检索算法的问题，是没有**闸门**。

天权不是把记忆交给一个相似度排名就完事。每一段记忆在被交给 Mind 核用于生成回复之前，必须通过五道闸门。任何一道不通过，记忆就被抑制或剔除。

五道闸门不可关闭、不可绕过、不可运行时降级为弱校验。

## 第一章 · 闸门管道架构

### 1.1 管道位置

```
用户查询
  │
  ▼
[编码] 查询文本 → 32D 向量 + location_fingerprint + 当前时间 + 当前话题
  │
  ▼
[G1] 语义初筛 ── HNSW Top-K
  │ 候选: 50-200 条
  ▼
[G2] 时空一致性校验 ── location_fingerprint 比对 + P1/P2/P3 分级抑制
  │ 候选: 缩减至 10-50 条
  ▼
[G3] 仿生遗忘衰减 ── 钙化/强度/衰减曲线
  │ 候选: 权重调整
  ▼
[G4] 意图区分 ── 主动回忆 vs 被动闲聊
  │ 候选: 加载级别调整
  ▼
[G5] 话题壁垒 ── 跨话题抑制
  │ 候选: 最终集
  ▼
[Mind 核] 上下文组装 + LLM 生成
```

### 1.2 管道纪律

```
🔴 G1→G2→G3→G4→G5 严格顺序, 禁止跳级
🔴 任何一级不可关闭、不可绕过
🔴 每级的阈值可在配置文件中调整, 但不可 disable
🔴 每级失败有独立的降级策略, 不阻塞整条管道
```

## 第二章 · G1：语义初筛

### 2.1 算法

```
输入: 查询 32D 向量 queryVec, 检索上限 topK
算法:
  1. HNSW 索引搜索 queryVec, 返回 topK 个最近邻
  2. 对每个候选计算 cosine 距离
  3. cosine < 0.3 → 直接通过
  4. cosine ≥ 0.3 → 标记低相关, 放入候选队尾

配置参数:
  G1_COSINE_THRESHOLD = 0.3        # 硬通过阈值
  G1_DEFAULT_TOPK = 100            # 默认检索量
  G1_MAX_TOPK = 200                # 上限
  G1_MIN_CANDIDATES = 5            # 最少候选数 (不足时降低阈值重搜)
```

### 2.2 降级策略

```
场景 1: HNSW 索引损坏
  → 降级为关键词 LIKE 搜索 (原始数据层)
  → 结果标记 source=keyword_fallback
  → 向 aqc_records 写入告警

场景 2: 候选数量 < G1_MIN_CANDIDATES
  → 阈值临时放宽至 0.5, 重搜
  → 仍不足 → 返回现有结果, 标记 sparse_results
```

### 2.3 输出

```
G1Result {
  candidates: ScoredMemory[]     // 按 cosine 距离升序
  totalSearched: number          // HNSW 实际扫描量
  indexHealth: "ok" | "degraded" | "damaged"
}
```

## 第三章 · G2：时空一致性校验

### 3.1 算法

```
输入: G1 候选集, 当前 location_fingerprint
算法: 逐条比对

  1. 计算候选海胆的 location_fingerprint 与当前的 cosine 距离
  2. 分级判定:

     距离 ≤ 0.3 → PASS (全量加载)
       记忆与当前场景高度一致, 完整加载原文 + 32D 向量 + 实体标注

     0.3 < 距离 ≤ 0.6 → P1 抑制 (摘要模式)
       仅保留 L2/L3 摘要 + 实体标注, 屏蔽原文详细描述
       知识库条目只提取事实, 屏蔽场景原文

     0.6 < 距离 ≤ 0.8 → P2 抑制 (权重降级)
       全局权重 × 0.3, 退出主推理队列
       退入"参考记忆"池, 仅在无其他候选时使用

     距离 > 0.8 → P3 剔除
       直接移出候选集, 不可恢复

配置参数:
  G2_PASS_MAX      = 0.3     # PASS 上限
  G2_P1_MAX        = 0.6     # P1 上限
  G2_P2_MAX        = 0.8     # P2 上限
  G2_WEAK_MATCH    = true    # 瑶光空白期弱匹配开关
```

### 3.2 瑶光空白期兼容

```
当 location_fingerprint 为全 0 (TEMP_SCENE_COMPAT):
  → 打印告警日志: "G2: location_fingerprint 未就绪, 降级为弱匹配"
  → 所有候选标记为 PASS (不做区位过滤)
  → 向 aqc_records 写入兼容告警
  → 瑶光上线后关闭此分支
```

### 3.3 输出

```
G2Result {
  passed: ScoredMemory[]         // PASS 级
  summary_only: ScoredMemory[]   // P1 级 (仅摘要)
  degraded: ScoredMemory[]       // P2 级 (低权重)
  eliminated: number             // P3 剔除数量
  compat_mode: boolean           // 是否瑶光空白期兼容模式
}
```

## 第四章 · G3：仿生遗忘

### 4.1 算法

```
输入: G2 候选集 (passed + summary_only + degraded)
算法: 逐条计算衰减后权重

  1. 计算距今时间: elapsedHours = (now - lastAccessTime) / 3600000

  2. 计算衰减系数:
     λ = 0.01  (常态, 环境未切换)
     λ = 0.03  (环境切换: 当前 location_fingerprint 与记忆的时间分片 tag 跨越 >= 自然月分界)

  3. 衰减公式:
     decayedWeight = originalWeight × exp(-λ × elapsedHours)
     decayedWeight = max(decayedWeight, 0.05)  // 下限

  4. 钙化三级衰减 (独立运行, 每 24h):
     calcium ≥ 3.0 → calcium -= 0.02, strength ×= 0.995
     工作相关      → calcium -= 0.05, strength ×= 0.985
     中性          → calcium -= 0.10, strength ×= 0.95

  5. 回溯增强 (每次被 G3 选中):
     calcium += 0.2 (上限 10)
     strength += 0.05 × (1 - currentStrength)

配置参数:
  G3_LAMBDA_NORMAL     = 0.01   # 常态衰减系数
  G3_LAMBDA_ENV_CHANGE = 0.03   # 环境切换衰减系数
  G3_WEIGHT_FLOOR      = 0.05   # 权重下限
  G3_RECALL_BOOST      = 0.2    # 回溯增强量
  G3_STRENGTH_BOOST    = 0.05   # 强度增强系数
```

### 4.2 输出

```
G3Result {
  candidates: ScoredMemory[]     // 权重已调整
  decayed: number                // 本轮衰减条数
  boosted: number                // 本轮增强条数
  below_floor: number            // 低于下限的条数 (将被标记为冷数据)
}
```

## 第五章 · G4：意图区分

### 5.1 算法

```
输入: G3 候选集, 用户查询原文
算法: regex 意图检测

  1. 意图检测 (regex 匹配):
     主动回忆模式:
       关键词: 昨天|前天|上周|上个月|上次|记得|回忆|以前|曾经|那时候|那时
       人物:   妈妈|爸爸|老婆|同事|朋友|老板
       地点:   在公司|在家|上次去|那次在
       → intent = "active_recall"

     被动闲聊模式:
       其他所有情况
       → intent = "passive_chat"

  2. 加载策略:
     active_recall:
       PASS 级:    完整加载 (原文 + 32D + 实体 + 上下文)
       P1 级:      完整加载 (突破摘要限制)
       P2 级:      加载摘要 + 实体
     passive_chat:
       PASS 级:    加载 L2/L3 摘要 + 实体
       P1 级:      仅加载实体标注
       P2 级:      不加载

配置参数:
  G4_ACTIVE_RECALL_PATTERNS: 上述 regex 列表
  G4_MAX_MEMORY_CHARS_ACTIVE: 3000   # 主动回忆最大字符数
  G4_MAX_MEMORY_CHARS_PASSIVE: 800   # 被动闲聊最大字符数
```

### 5.2 输出

```
G4Result {
  intent: "active_recall" | "passive_chat"
  loadedMemories: LoadedMemory[]   // 按加载策略处理后的记忆
  totalCharCount: number
  loadLevel: "full" | "summary" | "entity_only"
}
```

## 第六章 · G5：话题壁垒

### 6.1 算法

```
输入: G4 候选集, 当前对话话题 (由对话组藤蔓判定)
算法:

  1. 话题聚类:
     候选集中的每条记忆提取其 locus_path + scene_tags
     基于 locus_path 前缀分组 (family.* / work.* / emotion.* …)

  2. 话题切换检测:
     当前对话话题 (过去 3 轮对话的主导 locus_path)
     vs
     候选记忆的 locus_path

     如果当前话题已切换 (前后 3 轮的 locus_path 前缀不同):
       前话题的记忆链 → 冻结 (权重 × 0.1), 不参与本次上下文组装
       新话题的记忆 → 正常加载

  3. 冻结规则:
     同一话题内的所有记忆保持原权重
     跨话题记忆: 权重 × 0.1 (冻结态)
     冻结态持续: 直到用户明确切换回此前话题

配置参数:
  G5_TOPIC_WINDOW         = 3       # 话题窗口 (前 N 轮)
  G5_FREEZE_WEIGHT        = 0.1     # 冻结权重系数
  G5_TOPIC_SWITCH_CONFIDENCE = 0.6  # 话题切换置信度阈值
```

### 6.2 输出

```
G5Result {
  active: LoadedMemory[]          // 当前话题记忆 (全权重)
  frozen: FrozenMemory[]          // 冻结记忆 (低权重, 暂不加载)
  currentTopic: string            // 当前主导话题
  previousTopic: string           // 前一话题
}
```

## 第七章 · 完整的闸门输出契约

### 7.1 最终输出结构

```typescript
interface FiveStageGateOutput {
  // G1 语义初筛
  semanticCandidates: number;        // HNSW 返回的原始候选数
  semanticLatencyMs: number;         // HNSW 检索耗时

  // G2 时空一致性
  passedCount: number;               // PASS 级
  p1Count: number;                   // 摘要级
  p2Count: number;                   // 低权重级
  p3Eliminated: number;              // 剔除级
  locationCompatMode: boolean;       // 瑶光空白期弱匹配模式

  // G3 仿生遗忘
  decayedWeightSum: number;          // 总衰减量
  recallBoostCount: number;          // 回溯增强条数

  // G4 意图区分
  intent: "active_recall" | "passive_chat";
  totalMemoryChars: number;

  // G5 话题壁垒
  frozenCount: number;               // 冻结条数
  currentTopic: string;

  // 最终结果
  finalMemories: LoadedMemory[];     // 通过全部五级的记忆
  gateTrace: GateTrace[];            // 每级的执行日志
}
```

### 7.2 管道失败降级矩阵

| 失败场景 | 降级策略 | 对用户的影响 |
|---------|---------|------------|
| G1 HNSW 损坏 | 关键词 LIKE 搜索 | 语义联想能力丧失, 精确关键词仍可用 |
| G2 瑶光未就绪 | 弱匹配 (全 PASS) | 时空过滤关闭, 可能场景串扰 |
| G2 区位指纹损坏 | 该海胆标记 PASS, 日志告警 | 单个海胆时空过滤跳过 |
| G3 衰减数据损坏 | 使用默认权重 (0.5) | 遗忘功能暂停 |
| G4 意图分类不确定 | 默认 passive_chat | 记忆加载偏保守 |
| G5 话题判定不确定 | 不冻结任何记忆 | 话题壁垒暂停 |
| 全管道超时 (> 2s) | 返回 G1 Top-5, 跳过 G2-G5 | 降级为简单检索 |

## 第八章 · 监控告警

### 8.1 每级门禁监控指标

```
G1: HNSW 索引健康度, 每次检索耗时 (P50/P99)
G2: P3 剔除比例 (> 50% 告警), location_fingerprint 覆盖率
G3: 衰减率趋势, 低于下限率
G4: 主动/被动比例, 误判日志
G5: 冻结记忆数量, 话题切换频率
```

### 8.2 全局告警阈值

```
G2_P3_RATIO_HIGH      > 0.5   → 可能区位指纹异常, 通知瑶光域
G3_DECAY_RATE_HIGH     > 0.3   → 记忆衰减过快, 检查衰减参数
G1_LATENCY_P99_HIGH    > 500ms → HNSW 索引可能碎片化, 触发重建
FULL_PIPELINE_TIMEOUT  > 2000ms → 降级为简单检索
```

## 附录 A · 与现有 MemoryGate 的关系

现有 `wenstar-cc/src/app/conversation/MemoryGate.ts` 是一个 4 模式分类器 (casual/memory_recall/vague_recall/knowledge_query)，不等同于本规范定义的五级闸门。它是 G4（意图区分）和 G5（话题壁垒）的雏形。迁移路径：

- MemoryGate 的 4 模式分类 → 保留, 整合入 G4 intent 判定逻辑
- MemoryGate 的 hallucination guard → 保留, 作为 G5 后置校验
- MemoryGate 的位置 → 从 chat.ts 中独立出来, 挂载到天权底座第三层
