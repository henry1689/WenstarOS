# 天权与 MSA 融合互补方案

**日期**: 2026-07-10  
**核心问题**: 天权（外部认知 OS）与 MSA（模型内部注意力机制）如何取长补短、协同工作  
**前置说明**: MSA = Memory Sparse Attention（阿里通义团队，模型层注意力替换机制），天权 = 独立认知记忆 OS（两者不在同一层，不冲突、不替代、可互补）

---

## 一、核心认知：层级不同，不存在替代关系

这是最关键的起点——天权和 MSA 不在同一层，不构成非此即彼的选择：

```
                    ┌─────────────────────────┐
                    │     用户交互层            │
                    │   WebUI / 语音 / API     │
                    └──────────┬──────────────┘
                               ↓
┌──────────────────────────────────────────────────┐
│               天权 · 认知 OS                      │
│  （外部独立运行，不绑定模型）                      │
│                                                   │
│  ┌──────────┐  ┌──────────┐  ┌────────────────┐  │
│  │ 36D 感知   │  │ 五级闸门  │  │ DNA 编码总线    │  │
│  │ 采集融合   │→│ 时空过滤  │→│ + L0-L3 蒸馏   │  │
│  └──────────┘  └──────────┘  └───────┬────────┘  │
│                                       │           │
│                               组装好的轻量化上下文    │
│                               (L2/L3 摘要 + 状态)  │
└───────────────────────────────────┬──────────────┘
                                    ↓
┌──────────────────────────────────────────────────┐
│           MSA 增强型 LLM（模型层）                 │
│                                                   │
│  ┌────────────────────────────────────────────┐   │
│  │  MSA Memory Sparse Attention               │   │
│  │  • 双路由稀疏寻址（主题级+Token级）          │   │
│  │  • 100M token 上下文，线性算力              │   │
│  │  • 短训长推（64K→100M 零样本外推）          │   │
│  │  • Memory Interleave 多跳推理              │   │
│  └────────────────────────────────────────────┘   │
│                    ↕                              │
│            LLM 权重（4B/7B/13B...）                │
└──────────────────────────────────────────────────┘
                    ↓
                生成回复
```

**MSA 管的是「模型一次性能看多长、多快」。**  
**天权管的是「模型应该看什么、记住什么、怎么成长」。**

两者完全可以同时存在，且效果是乘法叠加的。

## 二、各自的定位边界

| 能力 | MSA | 天权 | 关系 |
|------|-----|------|------|
| 百毫秒级检索亿级记忆 | ❌ 无（模型内部无外部存储） | ✅ 双路由 + Zvec | 天权提供素材 |
| 单次推理看 100M tokens | ✅ 线性算力 | ❌ 不负责推理 | MSA 提供算力保障 |
| 记忆多维认知（36D） | ❌ 纯文本 KV | ✅ 仿生全维度 | 天权独有 |
| 时空一致性约束 | ❌ 无概念 | ✅ 五级闸门 P1-P3 | 天权独有 |
| 记忆动态演化 | ❌ 静态 KV | ✅ 巩固/衰减/归档 | 天权独有 |
| 零样本超长外推 | ✅ Doc-RoPE | ❌ 不介入 | MSA 独有 |
| 记忆/知识统一过滤 | ❌ 无 | ✅ ZVEC + 五级闸门 | 天权独有 |
| 多跳迭代推理 | ✅ Memory Interleave | ⚠️ 需要外部实现 | 可互相补充 |
| DNA 全链路溯源 | ❌ 无 | ✅ 全局编解码器 | 天权独有 |
| 人格演化 | ❌ 无 | ✅ 每日 L3 画像更新 | 天权独有 |

## 三、融合方案：天权做「选材」，MSA 做「烹饪」

### 3.1 天权输出 = MSA 的上下文输入

这是最自然的融合点：

```
用户输入
  ↓
天权：
  1. 36D 感知融合 → 理解当前状态
  2. 记忆检索 + ZVEC 检索 → 五级闸门过滤
  3. L0-L3 分层 → 组装轻量化上下文（L2/L3 摘要）
  4. DNA 编码 → 输出标准化的「记忆包」
  ↓                          ← 天权输出给模型的不是原始数据，
天权输出 → MSA 增强型 LLM     而是经过时空校验的、有序的、摘要化的上下文
  ↓
MSA 处理：
  1. 接收天权输出的上下文（约 10K-50K tokens）
  2. MSA 稀疏注意力计算（线性复杂度，不因上下文增大而变慢）
  3. Memory Interleave：如果需要多跳推理，发起二次检索请求至天权
  4. 生成回复
  ↓
  返回用户
```

### 3.2 关键接口定义

```typescript
// 天权 → MSA 的标准化上下文包
interface TianquanContext {
  // 当前状态摘要（天权第一层产出）
  currentState: {
    perception36d: number[];       // 本轮 36D 感知快照
    dominantDrive: string;         // 当前主导趋力
    location: Buffer;              // 当前区位指纹
    timestamp: number;             // 当前时间
  };

  // 记忆检索结果（天权第二层产出，已通过五级闸门）
  memoryContext: {
    l2_summaries: string[];        // 相关场景摘要（日常闲聊）
    l3_profile: string;            // 人格画像
    // 主动回忆时额外包含：
    l1_facts?: string[];           // 原子事实
    l0_excerpts?: string[];        // 原始对话片段（仅复盘）
  };

  // ZVEC 知识库检索结果（已通过五级闸门）
  knowledgeContext?: {
    relevantEntries: Array<{
      summary: string;
      source: string;
      sceneLabel?: Buffer;
    }>;
  };

  // 多跳推理回调（供 MSA Memory Interleave 调用）
  iterativeRetrieval?: (expandedQuery: string) => Promise<ContextSupplement>;
}
```

### 3.3 Memory Interleave 的协同

MSA 的 Memory Interleave 是多跳迭代推理。当天权 + MSA 协同工作时：

```typescript
// 标准流程（无多跳）
Tianquan.retrieve(query) → 五级闸门 → context
  → MSA.generate(context) → reply

// 多跳流程（Memory Interleave 激活）
Round 1:
  Tianquan.retrieve(query) → context1
  → MSA.generate(context1) → 初步回答（附带置信度低）
  → MSA 向天权发起 expandedQuery

Round 2:
  Tianquan.retrieve(expandedQuery) → context2（更精准）
  → MSA.generate(context1 + context2) → 补充推理

Round 3+:
  继续迭代，直到 MSA 置信度达标
```

MSA 的 Memory Interleave 管的是**模型内部的多次注意力计算**。天权在这里提供的是**每次迭代时重新检索的更精准的上下文**——两者互补，缺一不可。

## 四、三档融合模式（按硬件能力分级）

### 模式 A：完整融合（有 GPU + MSA 模型）

```
天权（CPU/内存）←→ MSA 增强型 LLM（GPU）
  天权产出 L0-L3 摘要上下文
  → MSA 接收并做稀疏注意力推理
  → 如有需要，MSA Interleave 回调天权做二次检索
```

适用：PC 游戏本、工作站、服务器  
效果：天权提供精准上下文 + MSA 提供百毫秒级 100M token 上下文窗口，**效果叠加最大化**

### 模式 B：天权自研双路由替代（无 MSA 模型）

```
天权（CPU/内存）
  天权自研双路由（路由 A 36D 粗筛 + 路由 B 语义精筛）
  + Zvec HNSW 索引
  → 天权组装上下文 → 标准 LLM
```

适用：无 GPU、嵌入式、低配设备  
效果：天权全权负责检索与上下文组装，LLM 只做推理

### 模式 C：混合（当前 Demo 可行路径）

```
天权自研双路由（阶段一）
  ↓ 天权产出的轻量化上下文
標準 LLM（无 MSA，现有 DeepSeek API）
```

这就是当前 P0-P1 规划的状态。未来升级到 GPU 设备后，将标准 LLM 替换为 MSA 增强型 LLM，天权侧代码无需任何改动——**接口不变，底层模型可替换**。

## 五、总结：天权 + MSA = 完整记忆认知闭环

```
MSA 让模型「看得更长」     —— 模型层能力
天权 让模型「看得更准」    —— 认知层过滤
天权 让系统「记得住、会成长」—— OS 层记忆管理

两者组合：
  天权负责：该记什么、该想什么、怎么过滤、怎么演化
  MSA 负责：怎样在超长上下文中高效找到关键信息
  标准 LLM 负责：最终的自然语言生成

三层独立、各司其职、可插拔替换
```

**一句话**: 天权决定「给模型喂什么」，MSA 决定「模型怎么高效消化」。两者不替代、不冲突、不存在二选一。
