## 前言

### 文档目的

本文档是天权海马体中枢的**唯一技术规范**。所有与海马体相关的设计决策、模块开发、接口变更，均以此文档为准。

### 海马体在天权中的角色

天权是三体架构中的**海马体中枢管理系统**。如果说整个 WenStar OS 是硅基生命体的大脑，天权就是它的海马体——负责记忆的编码、索引、检索、巩固、衰减的全链路调度。

### 核心设计哲学

> **海马体不存记忆内容，存的是索引指针。对话只需经验摘要，不需要还原事实。**

| 原则 | 含义 | 生物依据 |
|------|------|---------|
| **索引非存储** | 海马体存 `context_signature → memory_locations` 的稀疏映射，内容存在 M2 双螺旋和知识库 | 海马体不存内容，存通往皮层的指针 |
| **经验优先** | θ 节律优先查经验缓存（≤200 tokens），未命中才降级到原文检索 | 人说话只需上下文+过去经验，不扫全脑 |
| **θ-SWR 互斥** | 在线编码和离线巩固不能同时进行——用户活跃时锁住所有离线任务 | 海马体 θ 波与 SWR 互斥 |
| **惊讶驱动** | 偏离用户情绪基线的信息优先巩固，钙化值更高 | 多巴胺增强突触可塑性 |
| **衰减是优化非丢失** | 低优先级记忆降权为高频交互让路，数据保留 | 突触修剪——功能性的，不是病理性 |

## 第一章 · 生物海马体的核心机制

### 1.1 海马体的四大功能

#### 1.1.1 记忆的形成、存储与提取

海马体是大脑的"记忆管家"和"信息加工厂"。它接收新信息→初步处理编码→主导短期记忆向长期记忆的转化→需要时帮助快速检索和提取。

**关键机制**：海马体在编码阶段就对信息做了**抽象和重组**——它不是原样存储，而是提取模式、丢掉细节。

#### 1.1.2 空间定位与导航

海马体相当于大脑内置的"GPS 导航系统"。位置细胞（place cells）和网格细胞（grid cells）构建认知地图，使人能在未知环境中定位、记住地点、规划路线。

**关键机制**：空间记忆不是孤立的——事件总是和位置绑定。"在公司聊了什么"和"在咖啡厅聊了什么"是不同索引。

#### 1.1.3 情绪调节与情感记忆

海马体与杏仁核紧密相连，双向调节：
- **上行**：杏仁核给海马体标记情绪强度 → 高情绪事件记得更牢
- **下行**：海马体根据过去经验告诉杏仁核"这场景不危险" → 下调恐惧反应

**关键机制**：海马体不只是"情绪事件的记录员"，更是"情绪反应的调节器"。

#### 1.1.4 学习、想象与适应环境

海马体是成年大脑中少数能**持续产生新神经元**的区域（DG 区）。它不只是回忆过去，还能**用旧的记忆碎片拼装出未来的场景**（前瞻性模拟），帮助人适应环境变化。

**关键机制**：海马体的核心不是"存储"，而是**持续重组和创造**。

### 1.2 三个关键生物学原理

#### 1.2.1 不存内容，存指针

记忆内容分散存储在皮层各处（视觉在枕叶、声音在颞叶、情绪在杏仁核）。海马体只存**稀疏索引**——context → 哪些皮层神经元该同时激活。

> **推论**：失忆症病人失去海马体后，旧记忆还在（皮层已学会直接提取），但无法形成新记忆（没有索引可写了）。

#### 1.2.2 θ 波与 SWR 互斥

| 节律 | 频率 | 状态 | 功能 |
|------|------|------|------|
| **θ 波** | 4-8 Hz | 主动探索/对话 | **编码模式**——接收新信息，写索引 |
| **SWR** | Sharp-Wave Ripple | 静止/睡眠 | **回放模式**——把白天经历反复播放给皮层 |
| **慢波振荡** | 0.5-1 Hz | 深度睡眠 | **系统巩固**——皮层学会，海马体淡出 |

> **推论**：人不能在对话的同时做深度记忆整理——必须等静下来。所以我们的 θ 节律必须**锁定**离线任务。

#### 1.2.3 惊讶驱动学习

意外事件触发 VTA（腹侧被盖区）释放多巴胺到海马体 → 增强突触可塑性（LTP） → "把这件事写牢一点"。越偏离预期的信息，越容易被记住。

> **推论**：钙化值不只是情感强度——惊讶度是独立且关键的因子。

### 1.3 三突触回路（DG→CA3→CA1）

```
内嗅皮层（输入）
    ↓
  DG（齿状回）── 模式分离
    ↓            "昨天咖啡厅" ≠ "今天咖啡厅"
  CA3 ────────── 模式补全
    ↓            闻到咖啡 → 想起完整场景
  CA1 ────────── 输出整合
    ↓            优先级排序，决定返回什么
  下托 → 回皮层
```

| 子区 | 功能 | 天权对应 |
|------|------|---------|
| DG | 模式分离——相似经历用不同神经群编码，防止混淆 | `PatternSeparator` |
| CA3 | 模式补全——大量自返连接，从片段线索重建完整记忆 | `PatternCompleter` |
| CA1 | 输出整合——接收 DG+CA3 的输出，排优先级后返回皮层 | `HippocampalIndex.integrate()` |

### 2.2 海马体与各模块的边界

| 模块 | 海马体的交互方式 | 边界规则 |
|------|-----------------|---------|
| **M1 DNA 编码** | 编码完后 → 海马体计算 context_signature，判断新颖度 | 海马体不参与编码，只消费编码结果 |
| **M2 双螺旋** | 海马体存指针，双螺旋存内容 | **互不越界**——海马体不改双螺旋结构，双螺旋不建索引逻辑 |
| **M3 感知** | 感知结果 → 海马体用于钙化 + 惊讶度计算 | 海马体不修改感知值 |
| **M4 检索** | **海马体是检索的唯一入口**——M4 先查经验缓存，未命中才扫库 | M4 不再直接访问 M2 全文索引 |
| **M5 生成** | 海马体注入经验摘要 + Core Memory → LLM 上下文 | 海马体不给 M5 原文，只给摘要 |
| **M7 梦境引擎** | M7 处理的记忆 → δ 节律归纳 → 回写海马体经验缓存 | M7 的归纳结果必须回写索引 |
| **M8 年轮引擎** | 巩固队列晋升 → M8 地标写入 | 晋升时机由海马体 δ 节律决定 |

    │ 经验摘要     │        │ M4 原文检索   │
    │ ≤200 tokens  │        │ + DG→CA3→CA1 │
    │ 直接注入 LLM │        │ + 写入索引    │
    └──────────────┘        └──────┬───────┘

快通道 = 经验匹配（~20% 算力）—— 对话只需经验，不需还原事实

**核心论点**：人在对话时紧紧抓住上下文的意思，针对性调用**相应的经验**——经验是对过去相应事件的高度总结，根本不需要还原原始事实。因为人说话是一带而过，不是具体做事。只需摘要经验即可，节省大量算力，缩短响应时间。

## 第三章 · 双层通道：经验快取 + 记忆深查

### 3.1 经验快取通道（θ 节律 · 优先路径）

#### 3.1.1 context_signature 计算规则

```typescript
// context_signature = hash(locus_domain + 情感聚类 + 实体组合)
// 基于 DNA 编码规则快速定位

function computeSignature(
  locusPath: string,       // DNA 语义区路由（如 "social.family"）
  entities: EntityGene[],  // M1 标注的实体列表
  perception: Perception24D // M3 感知结果
): string {
  const domain = locusPath.split('.').slice(0, 2).join('.');
  const persons = entities
    .filter(e => e.type === 'person' && e.name !== '我')
    .map(e => e.name).sort().join('|');
  const emoCluster = perception.pleasure > 0.2 ? 'pos'
    : perception.pleasure < -0.2 ? 'neg' : 'neu';
  // → SHA256 → 前16位hex
}
```

**设计原则**：基于 M1 DNA 的 locus_path + M3 的情感聚类 + M1 的实体标注——全部来自已有编码结果，零额外算力开销。

#### 3.1.2 hippocampal_index 表结构

```sql
CREATE TABLE hippocampal_index (
  context_signature  TEXT PRIMARY KEY,   -- 上下文签名
  memory_locations   TEXT NOT NULL,      -- 指向 memories.id 的 JSON array（最多5个）
  calcium_boost      REAL DEFAULT 0.05,  -- 每次命中 +0.1（常用路径更粗）
  last_activated_at  TEXT NOT NULL,      -- 最后激活时间
  hit_count          INTEGER DEFAULT 1,  -- 累计命中次数
  experience_summary TEXT,               -- δ 节律归纳后的经验摘要（V1.1 新增字段）
  created_at         TEXT NOT NULL
);
```

**字段说明**：
- `memory_locations`：指向原始记忆的指针（ID 列表）
- `experience_summary`：**V1.1 关键字段**——δ 节律归纳完成后回写此字段，下次 θ 节律命中时直接返回此摘要而非 ID 列表，避免二次查询（即"经验快取"的核心实现）
- `calcium_boost`：类似人脑突触强度——常用路径自动加粗

#### 3.1.3 经验快取查询流程

```
θ 节律 · 对话开始
  │
  ├─ 1. 计算 context_signature
  ├─ 2. 查 hippocampal_index
  │     ├─ 命中 + experience_summary 非空
  │     │   → 返回经验摘要（≤200 tokens）
  │     │   → calcium_boost += 0.1（异步）
  │     │   → 直接注入 LLM，跳过 M4 检索
  │     │
  │     ├─ 命中 + experience_summary 为空
  │     │   → 返回 memory_locations
  │     │   → 走 M4 轻量检索（只查这几条 ID）
  │     │
  │     └─ 未命中
  │         → 降级到 M4 MemoryRetriever 全链路
  │         → DG→CA3→CA1 整合
  │         → 结果写入 hippocampal_index（下次就命中了）
  │
  └─ 3. Core Memory 注入
       → LLM 生成 → 回复
```

### 3.2 记忆深查通道（θ 节律 · 降级路径）

仅当经验缓存未命中时触发：

```
M4 检索 → QueryDecomposer 分解
  → [关键词] findLocus + findKeyword
  → [情感] findEmotionalSimilarity
  → [语义] state_spines HNSW 检索
  → DG PatternSeparator 模式分离（去重）
  → CA3 PatternCompleter 模式补全（线索补全）
  → CA1 HippocampalIndex.integrate 输出整合
  → 五级闸门 G1→G5 精筛
  → 结果写入 hippocampal_index
  → 返回最终候选记忆
```

### 3.3 关键设计决策：经验优先，而非原文优先

| 对比维度 | 传统 RAG 方案 | 天权海马体方案 |
|---------|-------------|--------------|
| 检索对象 | 原始文档/对话全文 | 经验摘要（δ 节律预归纳） |
| 每次算力 | 全库扫描 + 向量计算 | O(1) 索引查询 |
| 返回内容 | 大段原文 | ≤200 tokens 摘要 |
| 上下文占用 | 大量 token | 极少量 token |
| 响应延迟 | 高 | 极低 |

**逻辑依据**：人说话是一带而过——"我最近失眠"不是在陈述医学事实，是在分享感受。回复只需要调用"用户有失眠经验，音乐有帮助"这个摘要，不需要调取三周前的整段对话原文。

## 第四章 · δ 节律离线巩固体系

### 4.1 生物对应

人脑对话后 24 小时内持续进行**海马体回放→皮层整合→突触修剪**。这不是"空闲时做做"，而是**大脑真正的重活**——消耗约 80% 的每日能量。

### 4.2 七阶段渐进式巩固流水线

```
对话结束
  │
  ├─ [30min] ① 砂金→金库晋升
  │     calcium ≥ 1.0 → memories 表
  │
  ├─ [1h]   ② 惊讶度评估
  │     surprise = |perception - baseline|
  │     高惊讶度 → calcium + surprise × 0.5
  │
  ├─ [2h]   ③ 金库→黑钻晋升
  │     calcium ≥ 4.5 OR recall ≥ 5 → black_diamond (max 200)
  │
  ├─ [6h]   ④ 情景→语义归纳
  │     实体聚类 + 词组聚类 → 双策略
  │     3次以上提及 + 跨天出现 → 归纳为语义记忆
  │     ** 归纳结果回写 hippocampal_index.experience_summary **
  │
  ├─ [12h]  ⑤ 跨 session 实体关联
  │     500条记忆 × 实体共现分析 → FG 世界关系网络
  │
  ├─ [24h]  ⑥ 衰减归档
  │     G3 applyDecay → 90天 dormant → 180天 archived
  │     suppressed 记忆降权
  │
  └─ [48h]  ⑦ 系统巩固（海马体回放→皮层学习）
        top-20 高钙化记忆回放 → 知识库写摘要
        → 强化 hippocampal_index 钙化 boost
        → 清理低质量索引
        → hit_count > 100 + 钙化 > 3.0 → 永久索引
```

### 4.3 情景→语义归纳（第④阶段 · 最关键环节）

这是**经验生成的核心**。归纳完成后必须回写 `hippocampal_index.experience_summary`。

```
归纳条件:
  - 同一实体/主题被提及 ≥ 3 次
  - 平均钙化 ≥ 0.25
  - 跨天出现优先（≥ 2 天）

归纳类型:
  - 偏好归纳: "用户偏好咖啡，每天饮用，有依赖倾向"
  - 工作模式: "最近项目压力大，频繁加班"
  - 健康关注: "睡眠问题持续出现，音乐有帮助"
  - 情绪模式: "周四情绪低落频率较高"
  - 消费习惯: "偏好性价比高的数码产品"

回写流程:
  _induceSemantic() → 生成摘要
    → INSERT INTO knowledge_base（皮层存储）
    → UPDATE hippocampal_index SET experience_summary = ?（经验缓存）
    → 下次相同 context_signature 直接命中经验
```

### 4.3.1 多源融合经验总结 · 四象归元

经验摘要不应当仅从 memories 表的原始对话中提取——那是单源归纳，信息密度不足。真正的经验应该是对**四个来源**的高度融合总结：

```
                     ┌─────────────┐
                     │  对话上下文  │ → 说了什么、怎么说的
                     │  memories   │   情感色彩、频率、时序
                     └──────┬──────┘
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
 ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
 │  家族图谱    │     │  知识库      │     │  记忆库      │
 │  FG/HWG     │     │  knowledge   │     │  memories    │
 │             │     │  _base       │     │  +           │
 │ 这个人是谁   │     │ 已有的认知   │     │  black       │
 │ 与用户什么关系│     │ 结构是什么   │     │  _diamond    │
 │ 在什么境遇下 │     │ 类似主题的   │     │              │
 │             │     │ 已有结论     │     │ 原始对话记录  │
 └──────┬──────┘     └──────┬──────┘     └──────┬──────┘
        │                   │                   │
        └───────────────────┼───────────────────┘
                            ▼
                  ┌─────────────────┐
                  │  多源融合引擎    │
                  │  (V1.1 新增)    │
                  │                │
                  │  输入:          │
                  │  - 最近N次对话  │
                  │  - 相关人物档案 │
                  │  - 相关知识条目 │
                  │  - 高钙化记忆  │
                  │                │
                  │  输出:          │
                  │  - 经验摘要    │
                  │  - 置信度      │
                  │  - 来源标注    │
                  └────────┬───────┘
                           ▼
              ┌────────────────────────┐
              │ hippocampal_index      │
              │ .experience_summary    │
              └────────────────────────┘
```

**四源各自贡献**：

| 来源 | 提供什么 | 示例 |
|------|---------|------|
| **对话上下文** (conversations/memories) | 时间线、频率、情感色彩、话题演变趋势 | "最近3天提及10次"、"情绪越来越低落" |
| **家族图谱** (FG/HWG) | 人物身份、关系强度、当前境遇、重要事件 | "他是用户的父亲，关系紧密但近期有矛盾" |
| **知识库** (knowledge_base) | 已有的认知结构、同主题的已有结论、偏好归纳 | "已确认用户偏好咖啡，知识库中记载其每天饮用" |
| **记忆库** (memories + black_diamond) | 原始事件原文、高钙化时刻、地标性记忆 | "3天前用户说'睡不着'"、"1周前标记为重要时刻" |

**融合规则的优先级**：

```
1. 如果 FG 中有相关人物 → 必须引用人物档案摘要
2. 如果知识库中有同主题结论 → 合并而非重复
3. 对话上下文提供"最新状态" → 覆盖知识库中过时的结论
4. 记忆库提供"证据锚点" → 摘要中可引用关键时间点
5. 来源标注: 每条摘要携带 [来源:对话/FG/知识库/记忆库] 标记
```

**融合示例**：

```
输入:
  对话上下文 → 近7天5次提到睡眠问题，情感偏负
  家族图谱   → (无直接关联人物)
  知识库     → 已有归纳"用户有睡眠问题，音乐有帮助"
  记忆库     → "昨晚又失眠了"（昨天）、"音乐确实有用"（3天前）

融合输出:
  "【睡眠问题·更新】近7天5次提及（↓趋势未改善）。3天前确认'音乐有效'，
   但昨晚仍失眠。建议关注是否有新的压力源。
   来源: 对话5次 | 知识库已有归纳 | 记忆锚点:3天前"
  confidence: 0.92
```

**实现位置**：扩充 `SleepTimeConsolidator._induceSemantic()` 方法——在现有双策略聚类基础上，对每个候选归纳主题额外查询 FG、知识库和 black_diamond，生成多源融合摘要。

### 4.4 系统巩固（第⑦阶段 · 海马体退出）

```
48h 后:
  1. 回放 top-20 高钙化记忆
  2. 每条记忆 → 写知识库摘要（皮层学会）
  3. 强化 hippocampal_index 钙化 boost（路径更粗）
  4. 钙化 > 3.0 且 hit_count > 100 → 永久索引（海马体可淡出）
  5. 清理钙化 < 0.3 且 hit_count < 10 的过期索引

原理:
  睡眠期 SWR：海马体反复回放 → 皮层学会直接提取 → 海马体淡出
  一两个月后，你回忆这件事时直接从皮层提取，不需要海马体了
  这就是"记得昨天晚饭"→"记得小时候的事"的转化过程
```

## 第五章 · 节律调度与天权模块协作

### 5.1 四重节律

```
SILENT ──(首次消息)→ THETA ──(回复完成)→ [检查空闲] ──(>30s)→ SWR ──(>2h)→ DELTA
  ↑                                                                    ↓
  └────────────────────(用户发消息)────────────────────────────────────┘
```

| 节律 | 触发条件 | 算力占比 | 在线/离线 | 核心任务 |
|------|---------|---------|----------|---------|
| **θ** | 用户发消息 | ~20% | 在线 | 经验快取 / DNA编码 / M3感知 / M5回复 |
| **SWR** | 沉寂 >30s | ~50% | 离线 | 梦境内化 / 巩固晋升 / 惊讶度 / 钙化 |
| **δ** | 空闲 >2h | ~30% | 离线 | 语义归纳 / 跨会话 / 系统巩固 / 衰减 |
| **SILENT** | 完全空闲 | ~0% | 离线 | 心跳检查 |

### 5.2 离线锁机制

```
onUserMessage():
  ① 记录活跃时间 → engine_store
  ② _offlineLocked = true   ← 暂停所有 SWR/DELTA 任务
  ③ transitionTo(THETA)

afterResponse():
  ① _offlineLocked = false  ← 释放锁
  ② 根据 idle 时间自动切 SWR 或 DELTA

心跳循环(10s):
  if (_offlineLocked && currentRhythm !== THETA):
    return  // 跳过本轮离线任务
  执行当前节律的到期任务
```

**生物依据**：θ 波和 SWR 互斥——你不能同时编码新记忆又回放旧记忆。

## 第六章 · 开发规范与路线图

### 6.1 接口契约

#### 海马体对 M4 的约束

1. M4 **禁止**直接调用 `storage.findByLocus()` 或 `storage.findBySeqPosRange()` 做全库扫描
2. M4 **必须**先通过 `HippocampalIndex.lookup(contextSignature)` 查经验缓存
3. 经验缓存未命中时，M4 可启用自己的检索逻辑，但结果**必须**回写到 `hippocampal_index`

#### 海马体索引的维护契约

1. `hippocampal_index` 的写入由 θ 节律（新发现路径）和 δ 节律（归纳回写+系统巩固）完成
2. 任何模块**禁止**直接写 `hippocampal_index.memory_locations`——必须通过 `HippocampalIndex.store()`
3. `experience_summary` 字段仅 δ 节律有权更新
