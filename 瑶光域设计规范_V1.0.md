## 序言 · 瑶光在三体中的角色

```
           ┌──────────────────────────┐
           │     太虚境 · 天权 (t)      │
           │     智脑 · 中枢调度        │
           └──────────┬───────────────┘
                      │
      ┌───────────────┼───────────────┐
      ▼               │               ▼
┌──────────┐          │          ┌──────────┐
│ 瑶灵 (l)  │          │          │ 瑶光 (g)  │
│ 硅基肉身   │          │          │ 客观世界   │
│ 纯主观体感  │          │          │ 纯客观参数  │
│ 不思考不记忆│          │          │ 不产生情绪  │
└──────────┘          │          └──────────┘
                      ▼
                 ┌──────────┐
                 │ 使用者 (人) │
                 └──────────┘
```

瑶光不是传感器、不是数据采集器——它是**完整的客观世界模型**。瑶光模拟的不是"摄像头看到了什么"，而是"这个世界客观上是什么样的"。

瑶光拥有五个计算单元，涵盖自然时空、空间建模、人体生理规律、人文社会规则、资源拓展。瑶灵的主观体感（32D）是建立在瑶光的客观基线之上的——没有瑶光提供的"标准心率是多少"、"标准皮质醇是多少"，瑶灵无法计算"这个人现在心率偏离了多少"。

### 1.2 单元 1：自然时空

```
功能: 提供当前时刻的客观时间/季节/天气/物候状态

输入: 系统时钟 + QWeather API + 农历计算引擎
输出: 天时上下文 (TemporalContext)
  ├── 公历时间: YYYY-MM-DD HH:mm:ss
  ├── 农历日期: 甲子年 六月初八
  ├── 节气: 夏至 后第3天
  ├── 月相: 上弦月, 照度52%
  ├── 自然季节: 夏 (小暑)
  ├── 时段: 午后 (12:00-18:00)
  ├── 物候: 蝉始鸣, 荷花盛开
  ├── 当前天气: 晴, 32°C, 湿度65%
  └── 日出/日落: 05:42 / 19:28

数据源:
  ├── TimeKeeper (系统时间, 唯一 Date.now() 来源)
  ├── CalendarEngine (农历/节气/节日)
  ├── LunarPhaseCalc (月相天文计算)
  ├── NaturalCycle (太阳/季节/子季节)
  ├── PhenologyTimeline (物候花卉/场景/食物)
  └── AmbientWeatherContext (QWeather API, 每30分钟刷新)
```

### 1.3 单元 2：空间建模

```
功能: 提供当前场景的客观空间参数 + 生成区位指纹

输入: 太虚境 SceneAnchor (位置/活动/裸露度) + 环境传感器 (未来)
输出: 空间上下文 (SpatialContext)
  ├── 场景类型: home.bedroom / office.desktop / public.transit / outdoor.park
  ├── 区域属性: 私密 / 半公开 / 公开
  ├── 物理尺寸: 面积、容积 (估算)
  ├── 在场人员: partner / family / colleague / stranger / none
  ├── 环境参数: 温度/光照/噪音/空气质量
  └── location_fingerprint: 128-bit 区位指纹

location_fingerprint 生成:
  输入: 场景类型 + 区域属性 + 环境参数 + 人员标签
  算法: 128-bit 哈希 (前64位 = 空间特征, 后64位 = 环境特征)
  输出: Buffer(16) → 提供给 DNA 编码层 → GlobalUID 区位段
  用途: G2 闸门时空一致性校验的唯一依据
```

### 1.4 单元 3：人体生理规律

```
功能: 提供人体客观生理基线——瑶灵 32D 计算的参照系

这是瑶光最核心的单元。瑶灵不是凭空产生"心率90"这个值的——
它先查瑶光的"标准心率是72"，然后根据当前状态计算偏离。

输出: 生理基线表 (PhysiologicalBaseline)

  维度      指标             健康零点        健康区间        危险阈值
  ─────────────────────────────────────────────────────────────
  D1       血乳酸            1.0 mmol/L      0.5-1.5          >2.5
  D2       VAS疼痛评分       0 分             0-2              ≥6
  D3       交感神经兴奋度     35%             25%-55%          >80%
  D4       晨间皮质醇         14 μg/dL        10-18            >25
  D4       多巴胺            120 pg/mL        100-150          <60
  D4       血清素            110 ng/mL        80-150           <50
  D6       BMR代谢率         基础(性别/年龄/身高/体重公式)
  D7       乳酸清除速率       1.2 mmol/h      0.9-1.5          <0.5
  D8       体表温度           36.5℃           36.0-37.2        >38.5
  D8       环境噪音           40 dB            30-50            >80
  D9       自尊评分           32 分            28-40            <20
  D11      SAS焦虑量表        ≤30 分          ≤35              ≥50
  D12      催产素             45 pg/mL         38-60            <20
  D15      亲密催产素         50 pg/mL         40-65            <18
  D26      褪黑素             30 pg/mL         22-50            <8
  D32      静息心率           66 次/分         60-72            <55 或 ≥90
  D32      收缩压             115 mmHg         110-120          ≥140
  D32      舒张压             73 mmHg          68-78            <60
  D32      皮质醇均值         12.5 μg/dL       10-15            >24
  D32      愉悦激素均值       130 pg/mL        ≥110            <70

数据源:
  ├── 医学文献基线 (中国成年人参考值)
  ├── 性别/年龄/身高/体重修正系数
  ├── 昼夜节律修正 (皮质醇早晨高、夜晚低)
  └── 季节修正 (冬季 BMR 略高)
```

### 1.5 单元 4：人文社会规则

```
功能: 提供社会关系/礼仪/角色的客观规则

输出: 社会规则库 (SocialRuleBase)
  ├── 社交距离: 亲密(0-45cm) / 个人(45-120cm) / 社交(120-360cm) / 公共(>360cm)
  ├── 礼仪规范: 称谓规则 / 敬语级别 / 禁忌话题
  ├── 家庭角色: 长辈/平辈/晚辈 对应的责任和义务
  ├── 职场层级: 上级/平级/下级 对应的沟通规范
  └── 场景行为约束: 工作场景禁止亲密行为, 公共场景禁止裸露

用途: 瑶灵 D15-D20 (圈层人际) 的计算参照——"伴侣在场时催产素基线是多少"
      取决于瑶光提供的社交距离规则
```

### 1.6 单元 5：资源拓展

```
功能: 允许太虚境下发"世界解锁"指令, 动态扩展瑶光的感知能力

解锁指令类型:
  ├── 新场景注册: 注册一个新的空间类型 (如 "医院病房")
  ├── 新生理指标: 注册一个新的医学指标 (如 "血糖")
  ├── 新社会规则: 注册一个新的社交场景规则 (如 "相亲")
  └── 新自然现象: 注册一个新的物候现象 (如 "沙尘暴")

约束:
  ├── 仅允许长期推演时开启动态 DAG
  ├── 常规时序强制静态 YAML
  └── 解锁操作记录 audit log
```

## 第二章 · 与瑶灵的对偶计算

### 2.1 三体对偶模型

```
每维度 D1-D32 的最终值 = 瑶灵主观体感 + 瑶光客观基线的对偶计算结果

计算流程 (每维):
  1. 瑶光提供客观基线: baseline_value (该维度在正常状态下的标准值)
  2. 瑶灵计算主观体感: sensation_value (该维度当前感受到的值)
  3. 太虚境天权融合仲裁:
     deviation = sensation_value - baseline_value
     偏离 = 主观体感 - 客观基线

     偏离 > 0 → 主观感觉优于客观基线 (更开心、更放松)
     偏离 < 0 → 主观感觉差于客观基线 (更难受、更紧张)
     |偏离| 越大 → 主观体验越强烈
     偏离接近 0 → 主观感受与正常状态一致
```

### 2.2 对偶计算示例

```
例 1: D4 皮质醇
  瑶光基线: 皮质醇 = 14 μg/dL (健康成人晨间标准值)
  瑶灵体感: 皮质醇 = 22 μg/dL (工作压力 + 缺乏陪伴 + 长期久坐)
  天权仲裁: deviation = 22 - 14 = +8
  → 皮质醇偏高 8 μg/dL, 处于风险区间
  → 32D 写入 value = 22, 同时绑定 baseline=14, deviation=+8

例 2: D12 催产素 (幸福感)
  瑶光基线: 催产素 = 45 pg/mL
  瑶灵体感: 催产素 = 38 pg/mL (伴侣陪伴 +15, 工作压力 -8, 孤独感 -14)
  天权仲裁: deviation = 38 - 45 = -7
  → 催产素偏低 7 pg/mL, 处于亚健康区间
  → 32D 写入 value = 38, 同时绑定 baseline=45, deviation=-7
```

## 第四章 · 工作流设计

### 4.1 wf_time_tick —— 时序推进

```
路由标签: time_tick
工作流限制: 常规静态; 长期演化可动态
触发: 定时器 (每 30 分钟) 或 用户请求即时查询

Phase 1: 采集天时数据
  ├── TimeKeeper.now() → 公历时间
  ├── CalendarEngine → 农历/节气
  ├── LunarPhaseCalc → 月相
  ├── NaturalCycle → 季节/子季节/时段
  ├── PhenologyTimeline → 物候
  └── AmbientWeatherContext → 天气

Phase 2: 组装天时快照
  └── UnifiedTemporalContext → promptBlock

Phase 3: 推送至太虚境
  └── TCP 总线 → tianquan_snapshot 频道
```

### 4.2 wf_scene_sim —— 场景模拟

```
路由标签: scene_sim
工作流限制: 常规静态; 长期推演可动态
触发: 用户位置变化 / 场景切换 / 太虚境 SceneAnchor 更新

Phase 1: 场景识别
  ├── 从太虚境 SceneAnchor 获取当前场景类型
  ├── 匹配空间建模单元 → 确定空间参数
  └── 检测场景切换 → 更新 location_fingerprint

Phase 2: 生成空间快照
  ├── 场景类型 + 区域属性 + 物理参数 + 在场人员
  └── location_fingerprint (128-bit)

Phase 3: 推送至太虚境
  └── TCP 总线 → tianquan_snapshot 频道
```

### 4.3 wf_world_snapshot —— 世界快照发射

```
路由标签: world_snapshot
工作流限制: 常规静态
触发: 每次用户交互时, 太虚境请求完整世界快照

Phase 1: 汇总五大单元
  ├── 自然时空 → 天时上下文
  ├── 空间建模 → 空间上下文 + location_fingerprint
  ├── 人体生理 → 生理基线表 (32 维)
  ├── 人文社会 → 社会规则库 (当前场景适用规则)
  └── 资源拓展 → (无新解锁时跳过)

Phase 2: 编码
  ├── 统一 SpineSnapshot 结构
  ├── Protobuf 序列化 + CRC32
  └── 绑定 dna_root_id

Phase 3: 发射至太虚境
  └── TCP 总线 → yaoguang_snapshot 频道
```

## 第五章 · 交互协议

### 5.1 上行通道（瑶光 → 太虚境）

```
频道: yaoguang_snapshot
内容: 32D 环境客观参数快照 + 天时上下文 + 区位指纹

消息格式:
  {
    cmd: "world_snapshot_push",
    payload: {
      global_uid: string,
      location_fingerprint: hex(128-bit),
      temporal_context: { ... },
      spatial_context: { ... },
      physiological_baselines: { dim_id: baseline_value, ... },
      scene_tags: string[],
      crc32: string
    }
  }
```

### 5.2 下行通道（太虚境 → 瑶光）

```
频道: yaoguang_snapshot (接收)
指令类型:
  ├── sample_config:    调整感知采样频率/精度
  ├── location_lock:    锁定/解锁空间区位追踪
  ├── scene_override:   手动设定场景标签
  ├── unlock_world:     解锁新世界规则 (单元 5)
  └── snapshot_request: 请求即时世界快照
```

## 第六章 · 开发铁律

```
🔴 瑶光仅产出客观参数, 不产生主观情绪/体感
🔴 瑶光提供的生理基线是"标准值", 不是"当前值"——当前值是瑶灵的工作
🔴 瑶光不存储持久化数据——所有快照推送至太虚境统一存储
🔴 常规时序强制静态 YAML; 长期环境推演允许开启动态 DAG
🔴 location_fingerprint 是 G2 闸门时空校验的唯一数据源
🔴 生理基线变更必须同步通知瑶灵更新阈值注册表
🔴 禁止 LLM 直接输出浮点值——所有生理基线来自医学文献硬编码
```
